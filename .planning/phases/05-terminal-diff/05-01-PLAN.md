# Plan 05-01: Terminal + Diff — Core Implementation

**Created:** 2026-06-15

## Objective

交付 Phase 5 的核心功能：内嵌终端面板（Rust portable-pty + Tauri Channel + xterm.js）和代码 Diff 展示（@git-diff-view/react），可交互的底部面板 UI。

## Tasks

### Wave 1: Rust PTY Backend (TRM-01, TRM-02)

#### Task 1.1: Add portable-pty dependency
- **Type:** setup
- **Files:** `apps/desktop/src-tauri/Cargo.toml`
- **Action:** 添加 `portable-pty = "0.8"` 到 Cargo.toml 的 `[dependencies]` 下
- **Verify:** `cargo check` 通过
- **Acceptance:** portable-pty crate 可被引用，无编译错误

#### Task 1.2: Create TerminalManager module
- **Type:** feature
- **Files:** `apps/desktop/src-tauri/src/terminal/mod.rs`
- **Action:** 创建 TerminalManager 结构体，管理多个 PTY 实例。包含：
  - `TerminalInstance` 结构体（id, shell_name, cwd, master, writer, shutdown_flag）
  - `spawn_terminal(shell, cwd, cols, rows)` → 创建 PTY + spawn shell
  - `write_to_terminal(id, data)` → 写入 PTY
  - `resize_terminal(id, cols, rows)` → 调整 PTY 大小
  - `close_terminal(id)` → 关闭 PTY
  - `close_all()` → 关闭所有 PTY
  - 自动检测系统 shell（macOS: /bin/zsh, Windows: powershell.exe）
  - PTY 崩溃自动重启（检测 reader EOF，重新 spawn）
- **Verify:** `cargo check` 通过，TerminalManager 结构体编译无错误
- **Acceptance:** TerminalManager 可创建/关闭 PTY 实例，读写 PTY 数据

#### Task 1.3: Implement Tauri Channel streaming commands
- **Type:** feature
- **Files:** `apps/desktop/src-tauri/src/terminal/commands.rs`
- **Action:** 创建 Tauri commands：
  ```rust
  #[tauri::command]
  fn spawn_terminal(channel: Channel<Vec<u8>>, shell: Option<String>, cwd: Option<String>) -> Result<String, String>
  ```
  - 创建 PTY，spawn channel reader 线程
  - Reader 线程从 PTY master 读取，通过 channel.send() 推到前端
  - 返回 terminal_id
  ```rust
  #[tauri::command]
  fn write_to_terminal(id: String, data: String) -> Result<(), String>
  ```
  - 写入数据到指定 PTY
  ```rust
  #[tauri::command]
  fn resize_terminal(id: String, cols: u16, rows: u16) -> Result<(), String>
  ```
  - 调整 PTY 窗口大小
  ```rust
  #[tauri::command]
  fn close_terminal(id: String) -> Result<(), String>
  ```
  - 关闭指定 PTY
- **Verify:** `cargo check` 通过，commands 函数签名正确
- **Acceptance:** commands 可被 Tauri 前端 invoke 调用

#### Task 1.4: Register commands and TerminalManager in Tauri builder
- **Type:** integration
- **Files:** `apps/desktop/src-tauri/src/lib.rs`
- **Action:**
  - 声明 `mod terminal;`
  - 在 `Builder::default()` 中使用 `.manage(TerminalManager::new())`
  - 在 `invoke_handler` 中注册 4 个 terminal commands
- **Verify:** `cargo build` 通过，Tauri app 可启动
- **Acceptance:** Rust 后端可接受前端 PTY 操作请求
