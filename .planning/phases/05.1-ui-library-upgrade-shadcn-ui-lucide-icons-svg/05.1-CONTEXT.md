# Phase 05.1: UI Library Upgrade - Context

**Gathered:** 2026-06-17
**Status:** Ready for planning

<domain>
## Phase Boundary

引入 shadcn/ui 组件库和 lucide-icons 图标库，替换手写 SVG 和自定义组件，统一前端组件体系。为 Phase 6 Agent Tools 开发提供标准化 UI 基础设施。

**在 Phase 05.1 内：** lucide-react 全局替换所有手写 SVG 图标、shadcn/ui init + 引入 Button/Sonner/ScrollArea/Sheet 组件、CSS 变量迁移至 shadcn OKLCH 色系、逐个替换现有组件
**不在 Phase 05.1：** 新功能开发、Agent Tools 集成（→ Phase 6）、业务组件重构

</domain>

<decisions>
## Implementation Decisions

### 替换策略与范围
- **D-01:** lucide-icons 全局替换所有手写 SVG 图标（Titlebar 窗口按钮、ThemeToggle、ChatToggle、StatusBar、各处零散 icon），减少约 80 行手写 CSS 内 SVG
- **D-02:** shadcn/ui 全量替换目标组件：Button（ChatToggle + ThemeToggle + Titlebar 按钮 + StatusBar）、Sonner（替代 ErrorToast）、ScrollArea（替代 ConversationList）、Sheet（替代 RightDrawer）
- **D-03:** 业务组件保留不动（MessageBubble、MessageList、WelcomePage、ChatView、TerminalPanel、DiffMessageCard），但规划/执行时可自由使用 shadcn 原子组件优化内部实现

### 样式主题融合
- **D-04:** 色系以 shadcn 默认 OKLCH（neutral/gray 色）为主，废弃现有 paladin-50~900 自定义蓝色阶
- **D-05:** shadcn `@theme inline` CSS 变量完全覆盖现有 `index.css` 中的 @theme 配置，导入 `tw-animate-css` 替代自定义 @keyframes
- **D-06:** 暗色模式兼容 — shadcn 和现有代码都使用 `.dark` class 策略，Zustand theme store 继续管理 HTML class 切换

### 引入顺序与风险
- **D-07:** 引入顺序 — Wave 1: lucide-react 安装 + 全局替换 SVG（风险最低，独立验证）→ Wave 2: shadcn init + 安装依赖 + CSS 变量迁移 → Wave 3: 逐个引入并替换组件（Button → Sonner → ScrollArea → Sheet）
- **D-08:** RightDrawer→Sheet 作为最高风险项 — 先行尝试替换，保留原 RightDrawer.tsx 作为备份。如 Sheet 布局/动画不兼容，回退保留原 RightDrawer
- **D-09:** 质量门槛 — `tsc --noEmit` + `biome ci` 0 errors + `vite build` 通过。样式一致性人工 review，不阻塞自动门

### Claude's Discretion
以下实现细节由 Claude 在规划/执行阶段自主决定：
- shadcn CLI init 的具体参数（`--style new-york`、`--baseColor neutral`）
- 各组件引入的具体 shadcn 组件名称和变体选择
- `cn()` utility 的具体实现位置（`src/lib/utils.ts`）
- lucide icon 名称映射（Minus/Maximize2/X/Sun/Moon/MessageSquare/Terminal/GitBranch）
- Sonner toast 的 placement 和 duration 配置
- ScrollArea 的 height/max-height 具体值
- Sheet 的 side/width 参数配置

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- **Zustand theme store** (`src/stores/theme.ts`): 暗色模式切换逻辑，shadcn 完全兼容
- **Tailwind @tailwindcss/vite** (`vite.config.ts`): 已配置，shadcn 原生支持
- **`@/` 路径别名** (`vite.config.ts`, `tsconfig.json`): 已配置，与 shadcn 默认一致

### Established Patterns
- Biome 单引号、2-space indent、no non-null assertions、a11y（SVG titles、button types）
- Zustand stores 在 `src/stores/`、components 在 `src/components/`
- `@layer base` 暗色模式策略：`@variant dark (&:where(.dark, .dark *))`

### Integration Points
- **Titlebar.tsx**: 3 个手写 SVG（MinimizeIcon/MaximizeIcon/CloseIcon）→ lucide `Minus`/`Maximize2`/`X`
- **ThemeToggle.tsx**: 手写 SVG 太阳/月亮 → lucide `Sun`/`Moon`
- **ChatToggle.tsx**: 自定义按钮 + 手写 SVG → `Button` variant="ghost" + lucide `MessageSquare`
- **ErrorToast.tsx**: 自定义 toast → Sonner `toast()`
- **ConversationList.tsx**: 自定义滚动列表 → `ScrollArea`
- **RightDrawer.tsx**: 自定义侧边抽屉 → `Sheet`
- **StatusBar.tsx**: 内联 button → `Button` variant="ghost" size="sm"

</code_context>
