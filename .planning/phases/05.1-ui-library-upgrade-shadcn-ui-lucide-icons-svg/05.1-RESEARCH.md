## 1. shadcn/ui + Tailwind v4 + Vite 兼容性

### 结论：原生兼容，无需 workaround

- shadcn/ui 最新版（`npx shadcn@latest init`）已原生支持 Tailwind v4 + Vite
- `@tailwindcss/vite` 插件无需降级或切换
- CLI 自动生成 `components.json`（`style: "new-york"`, `tsx: true`, `rsc: false`）
- **关键配置**（来自官方 Vite 安装指南 https://v3.shadcn.com/docs/installation/vite）：
  - `tailwind.config` 在 v4 中留空字符串
  - `tailwind.css` 指向 `src/index.css`
  - `aliases` → `@/components`, `@/lib/utils`, `@/components/ui`, `@/lib`, `@/hooks`

### 项目现状 vs 需求

| 配置项 | 当前 Paladin | shadcn 要求 | 操作 |
|--------|-------------|------------|------|
| `@tailwindcss/vite` | 已安装 | 需要 | 无需改动 |
| `vite.config.ts` path alias | `@` → `./src` | `@` → `./src` | 无需改动 |
| `tsconfig.json` paths | 已配置 | 需要 | 无需改动 |
| `@types/node` | 已安装 | 需要 | 无需改动 |
| `src/index.css` | `@import "tailwindcss"` + `@theme` | CSS variables | **覆盖替换** |

## 3. CSS 变量迁移方案

### 当前 index.css 结构

```css
@import "tailwindcss";
@variant dark (&:where(.dark, .dark *));
@theme { --color-paladin-50~900: ... }
body { @apply bg-white dark:bg-gray-900 text-gray-900 dark:text-gray-100; }
@keyframes slide-up { ... }
```

### 目标 shadcn 结构

```css
@import "tailwindcss";
@import "tw-animate-css";

@custom-variant dark (&:is(.dark *));  /* shadcn 暗色模式变体 */

:root { /* OKLCH CSS 变量 — 亮色 */ }
.dark { /* OKLCH CSS 变量 — 暗色 */ }

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --color-primary: var(--primary);
  /* ... */
  --radius-sm: calc(var(--radius) - 4px);
  --radius-md: calc(var(--radius) - 2px);
  --radius-lg: var(--radius);
}

@layer base {
  * { @apply border-border outline-ring/50; }
  body { @apply bg-background text-foreground; }
}
```

### 关键变更

1. **暗色模式变体**：`&:where(.dark, .dark *)` → `&:is(.dark *)`（shadcn 官方格式）
2. **色系**：paladin-50~900 蓝紫色废弃 → OKLCH neutral 灰色
3. **动画**：自定义 `@keyframes slide-up` → 替换为 `tw-animate-css` 提供的动画
4. **base layer**：`bg-white dark:bg-gray-900` → `bg-background`（通过 CSS 变量自动暗色）

## 5. 组件替换分析

### 5.1 Button 替换（低风险）

目标组件：ChatToggle、ThemeToggle、Titlebar 窗口按钮、StatusBar

**shadcn Button API：**
```tsx
import { Button } from '@/components/ui/button';

// variants: default | destructive | outline | secondary | ghost | link
// sizes: default | sm | lg | icon

<Button variant="ghost" size="icon"><Minus className="size-4" /></Button>
```

**替换策略：**
- `ChatToggle` → `<Button variant="ghost" size="icon" onClick={...}>`
- `ThemeToggle` → `<Button variant="ghost" size="icon" onClick={...}>`
- `Titlebar` 窗口按钮 → `<Button variant="ghost" size="icon">`
- `StatusBar` 终端切换 → `<Button variant="ghost" size="sm">`
- `ConversationList` 按钮 → `<Button variant="ghost" size="icon">` / `<Button variant="ghost" size="sm">`

### 5.2 Sonner 替换 ErrorToast（低风险）

**当前 ErrorToast：** 固定底部 fixed 定位、自定义 animate-slide-up 动画、emoji 图标

**Sonner API：**
```tsx
// main.tsx / App.tsx
import { Toaster } from '@/components/ui/sonner';

<Toaster position="bottom-center" />

// 使用
import { toast } from 'sonner';
toast.error('连接失败');

// 带操作
toast.error('连接失败', {
  action: { label: '重试', onClick: () => retry() },
});
```

**替换策略：**
- 移除 `ErrorToast.tsx` 组件 + `animate-slide-up` keyframe
- Root 层添加 `<Toaster />`（放在 App.tsx 或 main.tsx）
- 调用方从 `<ErrorToast message={...} onRetry={...} />` 改为 `toast.error(...)`
- 需要找到所有 ErrorToast 的调用点并替换

### 5.3 ScrollArea 替换 ConversationList 滚动容器（低风险）

**当前：** ConversationList 用原生 `overflow-y-auto` 滚动

**shadcn ScrollArea API：**
```tsx
import { ScrollArea } from '@/components/ui/scroll-area';

<ScrollArea className="h-[400px]">
  {items}
</ScrollArea>
```

**替换策略：**
- 用 `<ScrollArea>` 包裹对话列表内容
- 保留 ConversationList 内部逻辑（分组、重命名、删除），仅替换外层滚动容器
- 高度从 CSS class 或父容器自适应

### 5.4 Sheet 替换 RightDrawer（高风险）

**当前 RightDrawer：**
- 自定义 `div` + width 动画 + ResizeHandle
- 内部包含 TerminalPanel、TerminalTabBar
- 与 `useTerminalStore` 深度绑定
- 拖拽调整宽度（ResizeHandle）

**shadcn Sheet API：**
```tsx
import {
  Sheet, SheetContent, SheetHeader, SheetTitle,
} from '@/components/ui/sheet';

<Sheet open={isOpen} onOpenChange={setOpen}>
  <SheetContent side="right" className="w-[400px] sm:max-w-[600px]">
    <SheetHeader>
      <SheetTitle>Terminal / Diff</SheetTitle>
    </SheetHeader>
    {children}
  </SheetContent>
</Sheet>
```

**风险点：**
1. **宽度可拖拽** — Sheet 固定 width，不支持 ResizeHandle 动态调整。需要评估是否接受固定宽度或使用 CSS 变量控制
2. **内部交互** — Terminal + Tauri Channel 生命周期管理、Tab 栏交互在 Sheet 内需保持不变
3. **动画冲突** — RightDrawer 当前使用 `transition: width 200ms ease-in-out`，Sheet 使用 Radix Dialog 的 CSS 动画，方向不同
4. **焦点管理** — Sheet 有 focus trap，Terminal 的 xterm.js 需要焦点才能输入。可能需要 `modal={false}` 或焦点管理适配

**回退策略：** 保留 `RightDrawer.tsx` 不动，Sheet 替换仅在验证可行后执行。如失败，RightDrawer 保留原样。

## 6. Tauri 特殊性影响

### 对 shadcn/ui 的影响：无

- shadcn 组件使用标准 React + Radix UI，不依赖浏览器特定 API
- Tauri WebView（macOS WKWebView）完全支持
- 没有 SSR/hydration 问题（CSR only）

### 对 lucide-react 的影响：无

- 纯 SVG 组件，无浏览器 API 依赖

### 构建注意事项

- `vite build` 可能会有 `@radix-ui/react-dialog` 的 ESM/CJS 兼容问题（Sheet 引入时）
- 如果遇到，在 `vite.config.ts` 添加 `optimizeDeps`

## 8. 已知风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| Sheet 焦点 trap 导致 Terminal 无法输入 | MEDIUM | RightDrawer 功能降级 | `modal={false}` 或回退 |
| shadcn dark variant 与现有 `.dark` 策略冲突 | LOW | 暗色模式失效 | 统一为 `&:is(.dark *)` 格式 |
| OKLCH 色值与现有 paladin 色差异大 | LOW | 视觉效果变化 | 用户已确认接受 shadcn 色系为主 |
| biome ci 和 shadcn 组件风格冲突 | LOW | CI 阻塞 | `biome check --fix` 自动修复 |
| ErrorToast 调用方改造遗漏 | LOW | 编译错误 | tsc 类型检查覆盖 |
