# Phase 05.2 Research — Chat Area Redesign

## 1. CopilotChat v2 API 调研

### 1.1 CopilotChat 组件签名

```tsx
function CopilotChat({
  agentId?,       // Agent ID，默认 "default"
  threadId?,      // 线程 ID，不传则自动生成
  labels?,        // 部分文案覆盖
  chatView?,      // 自定义 ChatView 插槽 (SlotValue<typeof CopilotChatView>)
  isModalDefaultOpen?,
  attachments?,   // 多模态附件配置
  onError?,       // 作用域错误处理器
  throttleMs?,    // 消息更新节流
  ...props        // 透传到 CopilotChatView
})
```

**关键发现：**
- `CopilotChat` 内部调用 `useAgent()` 获取 agent 实例
- `...props` 透传到 `CopilotChatView`，包括 `className`、`messageView`、`input`、`scrollView`、`welcomeScreen` 等插槽
- `CopilotChat.View` 是 `CopilotChatView` 的别名，可单独使用

### 1.2 CopilotChatView 插槽系统

CopilotChatView 提供以下可覆盖插槽（通过 `renderSlot` 机制）：

| 插槽 | 默认组件 | 用途 |
|------|----------|------|
| `messageView` | `CopilotChatMessageView` | 消息列表渲染 |
| `input` | `CopilotChatInput` | 输入框 |
| `scrollView` | `CopilotChatView.ScrollView` | 滚动容器 |
| `suggestionView` | `CopilotChatSuggestionView` | 建议提示 |
| `welcomeScreen` | `CopilotChatView.WelcomeScreen` | 欢迎页 |
| `scrollToBottomButton` | `CopilotChatView.ScrollToBottomButton` | 回到底部按钮 |
| `feather` | `CopilotChatView.Feather` | 渐变遮罩 |

**布局关键：** 默认 ScrollView 内部已包含 `cpk:max-w-3xl cpk:mx-auto` 实现居中窄列布局。无需额外包裹。

### 1.3 className 传递

`CopilotChat` 的 `className` 通过 `...props` 透传到 `CopilotChatView` 的根 `<div>`。可直接用于外层布局控制。

### 1.4 CopilotChatView 静态子组件

```tsx
CopilotChatView.ScrollView       // 滚动容器（含 max-w-3xl mx-auto）
CopilotChatView.WelcomeScreen    // 欢迎屏
CopilotChatView.WelcomeMessage   // 欢迎消息
CopilotChatView.ScrollToBottomButton  // 回到底部按钮
CopilotChatView.Feather          // 渐变遮罩
```

## 2. useAgent Hook 调研

### 2.1 Agent 实例属性

```tsx
const { agent } = useAgent({ agentId? });

agent.messages      // Message[] — 当前线程消息列表
agent.isRunning     // boolean — Agent 是否正在运行
agent.threadId      // string — 当前线程 ID
agent.capabilities  // Agent 能力描述
agent.subscribe({...})  // 订阅状态变更
agent.runAgent({...})   // 触发 Agent 运行
```

### 2.2 可用于右侧工具栏的数据

- **消息列表** — `agent.messages` 包含所有消息，可统计 tool 调用
- **运行状态** — `agent.isRunning` 控制状态条显示
- **工具调用** — `message.toolCalls` 数组，每个 tool call 有 id/name/arguments
- **Token 用量** — CopilotKit v2 当前版本未暴露 token 级别统计（需自定义或从 AG-UI 事件提取）

### 2.3 useCopilotKit Hook

```tsx
const { copilotkit } = useCopilotKit();
copilotkit.agents           // 已注册 agent 列表
copilotkit.runtimeUrl       // 运行时 URL
copilotkit.headers          // 请求头
copilotkit.subscribe({...}) // 全局事件订阅
```

## 3. 侧边栏折叠方案

### 3.1 当前结构

```
App.tsx
  └─ <aside className="w-64 ...">  // 固定宽度 256px
       └─ <ConversationList />
     </aside>
```

### 3.2 方案：Zustand store + CSS 过渡

```tsx
// store: useChatStore 或新建 useUIStore
sidebarCollapsed: boolean
toggleSidebar: () => void

// App.tsx
<aside className={cn(
  "transition-all duration-200 overflow-hidden",
  sidebarCollapsed ? "w-0" : "w-64"
)}>
```

**展开按钮位置：** 对话区域左边缘的悬浮按钮，或 Titlebar 左侧。

### 3.3 与 CopilotSidebar 的关系

当前 ChatToggle 控制 CopilotSidebar（右侧浮层）。重构后：
- CopilotChat 将直接嵌入主区域（不再用 CopilotSidebar 浮层）
- CopilotSidebar 可保留或移除（取决于是否仍需要浮层模式）
- ChatToggle 按钮可改为控制左侧对话列表折叠

## 5. 右侧工具栏方案

### 5.1 布局

```
┌──────────────────────────────────┬──────────┐
│                                  │ 上下文    │
│        CopilotChat               │ · 模型    │
│        (居中窄列)                 │ · 线程ID  │
│                                  │ · 消息数  │
│                                  │          │
│                                  │ 工具调用  │
│                                  │ · [列表]  │
│                                  │          │
│                                  │ 快捷操作  │
│                                  │ · 清空    │
│                                  │ · 切换模型│
└──────────────────────────────────┴──────────┘
```

### 5.2 数据获取

- 模型信息：从 `useAgent` 或 agent 配置获取
- 消息数：`agent.messages.length`
- 工具调用：遍历 `agent.messages` 提取 `toolCalls`
- 清空对话：调用 `copilotkit` 或 agent 方法重置线程

## 6. 风险与约束

| 风险 | 影响 | 缓解 |
|------|------|------|
| CopilotChat v2 样式与 shadcn 冲突 | CSS 优先级问题 | CopilotKit 使用 `cpk:` 前缀，与 shadcn 不冲突 |
| Token 用量无法直接获取 | 右侧工具栏功能受限 | 先显示消息数/工具调用数，Token 留待 Phase 6 |
| CopilotSidebar 移除影响现有交互 | ChatToggle 行为变化 | 同步改造 ChatToggle 为侧边栏折叠控制 |
| CopilotChat 内置 Diff 渲染未知 | 当前 DiffMessageCard 可能失效 | 需验证 CopilotChat 的 markdown 渲染是否支持 diff 代码块 |
