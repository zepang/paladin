    - `from starlette.responses import StreamingResponse`
    - `import asyncio, json, uuid`

    新增端点函数 `@app.get("/approval/stream")`：

    1. 定义 `async def approval_stream(request: Request)`：
       - 创建本地 asyncio.Queue
       - 注册到全局 _approval_queues 列表（模块级变量：`_approval_queues: list[asyncio.Queue] = []`）
       - 实现内部 async generator：
         - 循环：从本地 queue 读取事件（asyncio.wait_for 15s 超时）
         - 有事件 → yield `data: {json.dumps(event)}\n\n`
         - 超时 → yield `: keepalive\n\n`（SSE 注释行，保持连接）
         - 检测 `await request.is_disconnected()` → break
       - finally 块从 _approval_queues 移除本地 queue

    2. 返回 StreamingResponse(event_generator(), media_type="text/event-stream")，headers 含 Cache-Control: no-cache, Connection: keep-alive, X-Accel-Buffering: no

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → POST /approval/{id} | 任何本地进程可发送 HTTP 回调——需防范伪造 |
| Server → SSE Client | 审批事件流经本地 HTTP 连接 |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-06 | Spoofing | POST /approval/{id} | medium | mitigate | request_id 为 UUID4（122-bit entropy）——攻击者需猜中有效 ID；不存在则返回 404 不泄露信息 |
| T-07-07 | Information Disclosure | SSE stream content | low | accept | localhost-only HTTP，不暴露到网络；事件仅含工具名和参数，不含敏感凭据 |
| T-07-08 | Denial of Service | SSE keepalive flood | low | accept | keepalive 注释行极小（`: keepalive\n\n`），15s 间隔不构成资源问题 |
| T-07-SC | Tampering | npm/pip/cargo installs | high | mitigate | slopcheck + blocking human checkpoint for pyautogui (Plan 04) |
</threat_model>
