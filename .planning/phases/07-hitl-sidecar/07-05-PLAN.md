    创建 ApprovalBridge.tsx — 审批事件桥接 + 状态管理层（per D-07 修订：自定义 hook，不使用 CopilotKit useHumanInTheLoop）：

    1. TypeScript 类型定义（文件顶部）：
       export interface ApprovalRequest { request_id: string; tool_name: string; args: Record<string, unknown>; reason: string; }
       export interface ApprovalContextValue { pendingApprovals: ApprovalRequest[]; current: ApprovalRequest | null; approve: (requestId: string) => Promise<void>; deny: (requestId: string) => Promise<void>; isConnected: boolean; }

    2. ApprovalContext = createContext<ApprovalContextValue | null>(null) — React Context 暴露给下游组件

    3. useApprovalContext() hook：const ctx = useContext(ApprovalContext); if (!ctx) throw new Error(...) — 确保在 Provider 内使用

    4. useApprovalBridge() hook（内部实现，供 ApprovalProvider 使用）：
       - useState 维护 pendingApprovals: ApprovalRequest[] 和 isConnected: boolean
       - useEffect 创建 EventSource('http://localhost:9876/approval/stream')
       - onmessage: 解析 event.data JSON → setPendingApprovals(prev => [...prev, parsed])（FIFO 追加到队尾）
       - onerror: 设置 isConnected=false，5s 后 setTimeout 自动重连（清除旧 EventSource，新建连接）
       - 组件卸载时 eventSource.close()
       - approve(requestId): await fetch(POST /approval/{requestId}, {decision:true}) → setPendingApprovals(prev => prev.slice(1))（移除队首）
       - deny(requestId): 同 approve，body {decision: false}
       - POST 失败时 toast.error('审批提交失败')（import { toast } from 'sonner'），不阻塞队列
       - current 为 computed: pendingApprovals.length > 0 ? pendingApprovals[0] : null

    5. ApprovalProvider 组件：export function ApprovalProvider({ children }) { const state = useApprovalBridge(); return <ApprovalContext.Provider value={state}>{children}</ApprovalContext.Provider>; }

    6. 导出：ApprovalProvider, useApprovalContext, ApprovalRequest, ApprovalContextValue
  </action>
