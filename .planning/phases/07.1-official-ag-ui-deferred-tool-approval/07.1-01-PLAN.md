<threat_model>
standard: ASVS Level 1
severity_gate: Block completion on any high severity finding.

Trust boundaries:
- Frontend approval UI: AguiApprovalInterrupt and ApprovalCard receive AG-UI interrupt data from CopilotKit runtime and display tool metadata to the user.
- CopilotKit resume payload: useInterrupt resolve() records { approved: boolean, reason?: string } entries and sends them back through AG-UI resume.
- HTTP boundary: /copilotkit receives RunAgentInput from the desktop runtime and passes the replayed request to AGUIAdapter.
- Backend adapter boundary: AGUIAdapter maps AG-UI resume entries to Pydantic AI deferred approval results.
- Agent tool boundary: create_deep_agent interrupt_on and Tool(..., requires_approval=True) gate write_file, execute, and Computer Use tools.
- Legacy fallback boundary: legacy_sse endpoints and callback store remain available but are not part of the official default path.
- Supply-chain boundary: Python dependency baseline relies on pydantic-ai[ag-ui], pydantic-deep, and transitive ag-ui-protocol.

Threats and required mitigations:
- High: Unauthorized tool execution if official mode also leaves ToolGuard approval callback active or omits interrupt_on. Mitigation: T1 tests assert interrupt_on mapping, empty ToolGuard.require_approval, and no approval_callback in agui_interrupt mode.
- High: Approval bypass if Computer Use tools are registered without native requires_approval. Mitigation: T1 tests assert Tool instances with requires_approval is True.
- High: Resume forgery or schema confusion if frontend sends legacy { decision: ... } payloads. Mitigation: T2 tests assert exact official payloads.
- Medium: Overbroad interrupt rendering could turn unrelated pauses into approval cards. Mitigation: T2 tests keep reason === "tool_call" predicate with negative cases.
- Medium: Local resume conversion drift could mis-map denial/approval semantics. Mitigation: T1 forbids DeferredToolResults conversion in /copilotkit and verifies body preservation.
- Medium: Supply-chain/package legitimacy drift could remove official AG-UI protocol support. Mitigation: T1 imports official protocol classes and checks dependency constraints/lockfile.
- Low: Sensitive tool args displayed in chat transcript. Mitigation: ApprovalCard truncates args and uses existing transcript context; do not add persistence or audit storage in this phase.
</threat_model>
