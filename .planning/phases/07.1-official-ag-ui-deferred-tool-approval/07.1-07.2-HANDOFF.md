# Phase 07.2 Handoff: Legacy SSE Approval Cleanup

Phase 07.1 keeps the official Pydantic AI AG-UI `interrupt/resume` approval path as the default while preserving `legacy_sse` as a selectable fallback. This handoff records cleanup targets for a future Phase 07.2; it does not remove, deprecate, or behaviorally change the fallback in Phase 07.1.

## Cleanup Boundary

Phase 07.2 may remove the retained legacy SSE approval path after Phase 07.1 verification passes and the official path is accepted as stable.

Cleanup candidates:

- Backend `GET /approval/stream` route.
- Backend `POST /approval/{request_id}` route.
- `apps/agent/src/agent/approval_store.py`.
- Legacy callback/store code in `apps/agent/src/agent/hitl.py`.
- `apps/desktop/src/components/approval/ApprovalBridge.tsx`.
- `apps/desktop/src/components/approval/ApprovalDialog.tsx`.
- Legacy approval tests that only cover the SSE/HTTP callback path.
- Old Phase 07 roadmap and context references to the SSE bridge.

## Retained In Phase 07.1

- `hitl.mode = "legacy_sse"` remains accepted.
- Legacy SSE source files remain available for fallback operation.
- Official AG-UI approval uses `AGUIAdapter.dispatch_request()` and CopilotKit `useInterrupt({ renderInChat: true })`.
- No audit log persistence, approval history expansion, schema migration, or cleanup implementation is included in Phase 07.1.

## Planning Notes

- Plan Phase 07.2 only after Phase 07.1 targeted backend and frontend approval tests pass.
- Verify no users still depend on `legacy_sse` before removing the routes or frontend bridge.
- Keep Computer Use behavior unchanged except for whichever approval path remains after cleanup.
