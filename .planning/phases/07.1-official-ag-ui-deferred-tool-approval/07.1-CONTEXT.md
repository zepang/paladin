# Phase 07.1: Official AG-UI Deferred Tool Approval - Context

**Gathered:** 2026-07-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 07.1 将本地已经完成的 official AG-UI deferred tool approval spike 收束为正式 GSD 交付：默认审批路径从 Phase 07 的自建 SSE/HTTP fallback 切换到 Pydantic AI 官方 `AGUIAdapter` interrupt/resume，前端通过 CopilotKit `useInterrupt(renderInChat: true)` 在聊天消息流里渲染审批卡片，并使用官方 `{ approved: boolean }` resume payload。

本期不重新定义 HITL 产品能力，不删除 legacy SSE fallback，也不修 unrelated `Titlebar` build errors。Phase 07.1 的工作是把官方路径固化、验证、提交；legacy cleanup 后续作为 Phase 07.2 处理。

</domain>

<spec_lock>
## Requirements (locked via SPEC.md)

**7 requirements are locked.** See `07.1-SPEC.md` for full requirements, boundaries, and acceptance criteria.

Downstream agents MUST read `07.1-SPEC.md` before planning or implementing. Requirements are not duplicated here.

**In scope (from SPEC.md):**
- Upgrade Agent dependencies to official `pydantic-ai[ag-ui]>=2.3.0` and `pydantic-deep>=0.3.34`.
- Remove Paladin's local AG-UI approval bridge from the default approval path.
- Delegate `/copilotkit` interrupt/resume handling to official `AGUIAdapter`.
- Route `agui_interrupt` approval-required tools through Pydantic Deep `interrupt_on`.
- Register Computer Use tools with native `Tool(..., requires_approval=True)` in `agui_interrupt` mode.
- Render official `tool_call` interrupts as approval cards in the CopilotKit chat stream.
- Use official `{ approved: boolean, reason?: string }` resume payloads.
- Preserve targeted backend and frontend tests for the official path.

**Out of scope (from SPEC.md):**
- Removing Phase 07 `legacy_sse` fallback — cleanup should be planned as Phase 07.2 after official AG-UI approval is verified.
- Deprecating `legacy_sse` — this phase keeps it as an available fallback without changing its lifecycle status.
- Rewriting `ApprovalBridge`, `ApprovalDialog`, `approval_store.py`, or `/approval/stream` — these belong to the legacy fallback path.
- Changing Computer Use behavior beyond approval routing — Phase 07 already owns screenshot/click/type behavior.
- Adding audit logs or approval history persistence — Phase 09 Admin Systems owns audit concerns.
- Fixing unrelated frontend `Titlebar` unused-variable build errors — they are known build blockers outside the approval path.

</spec_lock>

<decisions>
## Implementation Decisions

### Plan Shape
- **D-01:** Use a single consolidation plan for Phase 07.1: `07.1-01-PLAN.md`.
- **D-02:** The single plan should cover the already completed spike end-to-end: dependency upgrade, backend `AGUIAdapter` delegation, Agent approval wiring, frontend `useInterrupt` payload changes, bridge deletion, and verification.
- **D-03:** Do not split 07.1 into separate backend/frontend/verification plans. This phase is a formalization and validation pass over an already coherent migration, not a greenfield feature.

### Verification Scope
- **D-04:** Backend verification must include AG-UI dependency/protocol tests, `/copilotkit` dispatch/resume preservation tests, and Agent approval wiring tests.
- **D-05:** Frontend verification must include approval interrupt tests that cover `reason: "tool_call"` detection and official `{ approved: true/false }` resume payloads.
- **D-06:** A full frontend build may be run for visibility, but it only blocks 07.1 if failures are in 07.1 approval code. The known unrelated `Titlebar` unused-variable errors are recorded as an out-of-scope blocker, not a 07.1 failure.

### Legacy Fallback Handoff
- **D-07:** Keep `legacy_sse` selectable through `hitl.mode` during 07.1; do not remove it and do not mark it deprecated in this phase.
- **D-08:** Capture a follow-up Phase 07.2 cleanup handoff so later planning can remove legacy approval code without re-discovering the old path.
- **D-09:** Phase 07.2 cleanup candidates include `/approval/stream`, `/approval/{request_id}`, `apps/agent/src/agent/approval_store.py`, `apps/agent/src/agent/hitl.py` legacy callback/store code, `ApprovalBridge.tsx`, `ApprovalDialog.tsx`, legacy approval tests, and Phase 07 roadmap/context references to the old SSE bridge.

### the agent's Discretion
- The planner may choose exact pytest/vitest command grouping as long as D-04 and D-05 are covered.
- The planner may add a short manual inspection step for `npm run build` output if it helps document the `Titlebar` exception.
- The executor may keep code comments minimal and may avoid touching legacy fallback files unless needed to preserve compatibility after the official path changes.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `AGUIAdapter.dispatch_request()` in `apps/agent/src/server/main.py` is already the official AG-UI entrypoint for `/copilotkit`; 07.1 should keep this direct delegation.
- `_approval_mode()` in `apps/agent/src/agent/paladin_agent.py` already supports `agui_interrupt` and `legacy_sse`, which enables the fallback boundary.
- `ApprovalCard` is already reusable for chat-embedded approval display; `AguiApprovalInterrupt` adapts official interrupt data into that card.
- Existing backend and frontend tests already encode most 07.1 acceptance criteria.

### Established Patterns
- Agent configuration is read from `apps/agent/config/config.json`; `hitl.mode` should remain the switch for official vs legacy approval behavior.
- The Agent factory centralizes tool and capability registration in `create_paladin_agent()`.
- Frontend approval UI uses React components under `apps/desktop/src/components/approval/` and Vitest tests colocated under `__tests__`.
- GSD phase documents keep requirements in SPEC and implementation decisions in CONTEXT; do not duplicate locked requirements into the plan.

### Integration Points
- `/copilotkit` must preserve the request body, including `resume`, so official `AGUIAdapter` can translate resume entries.
- `create_deep_agent()` must receive `interrupt_on` for configured approval-required tools in `agui_interrupt` mode.
- Computer Use tools must be wrapped as `Tool(..., requires_approval=True)` only for the official path; legacy mode keeps the existing callback route.
- `useInterrupt` must be gated to official `tool_call` interrupts and must resolve official payloads.

</code_context>
