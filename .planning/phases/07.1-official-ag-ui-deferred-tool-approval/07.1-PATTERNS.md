### `apps/agent/src/server/main.py` - official AG-UI endpoint

**Analog:** `apps/agent/src/server/main.py:/copilotkit`

The official path should be a thin request-replay wrapper around `AGUIAdapter.dispatch_request()`. Copy this shape:

```python
from pydantic_ai.ui.ag_ui import AGUIAdapter

@app.post("/copilotkit")
async def copilotkit_endpoint(request: Request) -> Response:
    try:
        body = await request.json()
    except Exception:
        return JSONResponse({"detail": "Invalid JSON body"}, status_code=422)

    async def receive_once():
        return {
            "type": "http.request",
            "body": json.dumps(body).encode("utf-8"),
            "more_body": False,
        }

    replay_request = Request(request.scope, receive_once)
    return await AGUIAdapter.dispatch_request(
        request=replay_request,
        agent=agent,
        deps=getattr(agent, '_default_deps', None),
    )
```

Pattern notes:
- Preserve the full incoming JSON body, including `resume`, by replaying it into a Starlette `Request`.
- Do not locally parse `resume` or build `DeferredToolResults` in this endpoint.
- Keep legacy `/approval/stream` and `/approval/{request_id}` code untouched in this phase; it remains the `legacy_sse` fallback.
- Keep response style consistent with existing endpoints: `JSONResponse` for validation errors and official adapter response passthrough for AG-UI SSE.

### `apps/agent/src/agent/paladin_agent.py` - approval wiring

**Analog:** `apps/agent/src/agent/paladin_agent.py:_approval_mode()` and `create_paladin_agent()`

```python
def _approval_mode(raw_config: dict) -> str:
    mode = raw_config.get("hitl", {}).get("mode", "agui_interrupt")
    if mode not in {"agui_interrupt", "legacy_sse"}:
        raise ValueError(f"Unsupported hitl.mode: {mode}")

```

Use this approval split before `ToolGuard` and `create_deep_agent()`:

```python
if mode == "agui_interrupt":
    interrupt_on = {tool_name: True for tool_name in require_approval}
    guarded_require_approval: list[str] = []
    approval_callback = None
    approved_computer_tools = [
        Tool(tool, requires_approval=True) for tool in computer_tools
    ] if computer_tools else None
else:
    interrupt_on = None
    guarded_require_approval = require_approval
    approval_callback = create_approval_callback(timeout=timeout)
    approved_computer_tools = computer_tools if computer_tools else None
```

Then pass both the guard and official interrupt map into the existing factory call:

```python
guard = ToolGuard(**guard_kwargs)

agent = create_deep_agent(
    model=primary_model,
    system_prompt=instructions,
    capabilities=[guard],
    tools=approved_computer_tools,
    interrupt_on=interrupt_on,
    backend=backend,
)
```

Pattern notes:
- Official mode: approval-required configured tools flow through `interrupt_on`; `ToolGuard.require_approval` is empty and no `approval_callback` is installed.
- Legacy mode: keep `ToolGuard.require_approval` and `create_approval_callback(timeout=timeout)`.
- Computer Use names are appended to the approval list when tools load, then wrapped as `Tool(..., requires_approval=True)` only in `agui_interrupt` mode.
- Preserve existing agent attributes after factory creation: `_default_deps`, `_model_configs`, and `_hitl_sse_queue`.

### `apps/desktop/src/components/approval/AguiApprovalInterrupt.tsx` - CopilotKit interrupt adapter

**Analog:** `apps/desktop/src/components/approval/AguiApprovalInterrupt.tsx`

Keep this file as a thin adapter around `useInterrupt` and `ApprovalCard`:

```tsx
import { useInterrupt, type InterruptEvent } from '@copilotkit/react-core/v2';

export function isApprovalRequired(event: Pick<InterruptEvent, 'value'>): boolean {
  return isRecord(event.value) && event.value.reason === 'tool_call';
}

export function buildApprovalResumePayload(
  decision: 'approved' | 'denied',
): Record<string, unknown> {
  if (decision === 'approved') {
    return { approved: true };
  }
  return {
    approved: false,
    reason: 'User denied this tool call.',
  };
}
```

Mount the hook with inline chat rendering:

```tsx
export function AguiApprovalInterrupt() {
  useInterrupt({
    enabled: isApprovalRequired,
    renderInChat: true,
    render: ({ event, resolve }) => (
      <ApprovalInterruptRenderer event={event} resolve={resolve} />
    ),
  });

  return null;
}
```

Pattern notes:
- Gate only on `reason === 'tool_call'`; do not accept legacy `approval_required`.
- Keep helper functions exported for Vitest coverage.
- Read both camelCase and snake_case fields only as defensive metadata normalization, not as a protocol rewrite.
- Resolve with official `{ approved: boolean, reason?: string }` payloads.
