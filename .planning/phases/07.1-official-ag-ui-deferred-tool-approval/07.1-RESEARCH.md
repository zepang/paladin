## Official Behavior

- Pydantic AI's AG-UI docs show a Starlette/FastAPI route can directly return `AGUIAdapter.dispatch_request(request, agent=agent)`, so Paladin should keep `/copilotkit` as direct official adapter delegation. [CITED: https://pydantic.dev/docs/ai/integrations/ui/ag-ui/]
- Pydantic AI's AG-UI integration receives AG-UI messages as `RunAgentInput`, converts them to Pydantic AI types, runs the agent, and streams AG-UI events back as SSE. [CITED: https://pydantic.dev/docs/ai/integrations/ui/ag-ui/]
- Pydantic AI's AG-UI API documents `AGUIAdapter.deferred_tool_results` as the built-in translation from `RunAgentInput.resume[]` to `DeferredToolResults`; approval is deny-by-default unless `payload.approved == True`. [CITED: https://pydantic.dev/docs/ai/api/ui/ag_ui/]
- Pydantic AI maps `status == "cancelled"` to `ToolDenied("Cancelled by user.")`, maps `payload.approved is True` to `ToolApproved()`, supports `payload.editedArgs` as `ToolApproved(override_args=...)`, and treats malformed or false payloads as denial. [CITED: https://pydantic.dev/docs/ai/api/ui/ag_ui/]
- Pydantic AI deferred tools support a stop-the-world flow where an agent run ends with `DeferredToolRequests`, the caller gathers approvals/results, and the next run resumes with `DeferredToolResults`. [CITED: https://pydantic.dev/docs/ai/tools-toolsets/deferred-tools/]
- CopilotKit `useInterrupt` with standard AG-UI flow receives interrupts from `RUN_FINISHED` when `outcome.type === "interrupt"` and exposes `interrupt`, `interrupts`, `resolve(payload?, interruptId?)`, and `cancel(interruptId?)`. [CITED: https://docs.copilotkit.ai/langgraph-python/human-in-the-loop/useInterrupt]
- CopilotKit documents `renderInChat: true` as rendering the interrupt UI inline in the chat transcript. [CITED: https://docs.copilotkit.ai/langgraph-python/human-in-the-loop/useInterrupt]
- CopilotKit documents `resolve({ approved: true })` as recording a resolved resume entry and submitting `resume` back to the agent. [CITED: https://docs.copilotkit.ai/langgraph-python/human-in-the-loop/useInterrupt]
- CopilotKit standard interrupt objects have shape `{ id, reason, message?, toolCallId?, responseSchema?, expiresAt?, metadata? }`; Paladin should gate on `reason: "tool_call"` and pass through metadata to the existing card. [CITED: https://docs.copilotkit.ai/langgraph-python/human-in-the-loop/useInterrupt] [VERIFIED: codebase]

## Implementation Guidance

1. Plan a verification-first consolidation, not a redesign. The code already matches R1-R7, so the plan should formalize exact checks and commit hygiene. [VERIFIED: codebase]
2. Keep `/copilotkit` body replay intact. Do not reintroduce Paladin-side parsing from `resume` to `DeferredToolResults`; official `AGUIAdapter` owns that conversion. [CITED: https://pydantic.dev/docs/ai/api/ui/ag_ui/] [VERIFIED: codebase]
3. Keep `agui_interrupt` as default. Do not change the accepted `hitl.mode` values beyond `agui_interrupt` and `legacy_sse`. [VERIFIED: codebase]
4. Keep `ToolGuard` in the capabilities list for blocked tools and `legacy_sse`, but keep `ToolGuard.require_approval == []` and `approval_callback is None` in `agui_interrupt` mode. [VERIFIED: codebase]
5. Keep Computer Use approval in official mode by wrapping each Computer Use function with `Tool(..., requires_approval=True)`. Pydantic AI treats tools registered with `requires_approval=True` as requiring human approval. [VERIFIED: codebase]
6. Keep frontend approval rendering in `AguiApprovalInterrupt`, not the legacy `ApprovalBridge`/`ApprovalDialog` path. [VERIFIED: codebase]
7. Gate frontend rendering to official `tool_call` interrupts; arbitrary interrupts must not render as approval cards. [CITED: https://docs.copilotkit.ai/langgraph-python/human-in-the-loop/useInterrupt] [VERIFIED: codebase]
8. Use `resolve({ approved: true })` for approval and `resolve({ approved: false, reason })` for denial. Pydantic AI's AG-UI mapping explicitly treats only `payload.approved == True` as approval. [CITED: https://pydantic.dev/docs/ai/api/ui/ag_ui/] [VERIFIED: codebase]

## Risks And Pitfalls

- **Legacy fallback drift:** Keeping `legacy_sse` selectable means old routes, queues, callbacks, and frontend bridge code remain in the tree; plan verification should ensure official-mode changes do not break fallback selection, but cleanup belongs to Phase 07.2. [VERIFIED: codebase]
- **Double approval paths:** Accidentally leaving configured tools in both `interrupt_on` and `ToolGuard.require_approval` would create ambiguous behavior; tests should assert official mode clears the guard approval list and callback. [VERIFIED: codebase]
- **Resume schema regression:** A frontend regression to legacy `{ decision: ... }` payloads would be denied by official Pydantic AI mapping because approval requires `payload.approved == True`. [CITED: https://pydantic.dev/docs/ai/api/ui/ag_ui/] [VERIFIED: codebase]
- **Overbroad interrupt rendering:** Rendering every interrupt as an approval card could hijack non-approval AG-UI pauses; gate on `reason: "tool_call"`. [CITED: https://docs.copilotkit.ai/langgraph-python/human-in-the-loop/useInterrupt] [VERIFIED: codebase]
- **Build noise:** Full desktop build may fail on unrelated `Titlebar` unused-variable errors; treat that as non-blocking only if approval-specific tests pass and the errors are documented as outside 07.1. [VERIFIED: codebase]
- **Version assumptions:** Installed local packages match the intended minimums today, but the planner should include the dependency probe test because AG-UI interrupt/resume APIs are version-sensitive. [VERIFIED: codebase]
