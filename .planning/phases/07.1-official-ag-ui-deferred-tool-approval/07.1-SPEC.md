# Phase 07.1: Official AG-UI Deferred Tool Approval — Specification

**Created:** 2026-07-02
**Ambiguity score:** 0.19 (gate: ≤ 0.20)
**Requirements:** 7 locked

## Goal

Paladin 将工具权限审批的默认路径从 Phase 07 的自建 SSE/HTTP fallback 切换为官方 Pydantic AI AG-UI interrupt/resume deferred tool approval，并在 CopilotKit 对话消息流内渲染审批卡片。

## Background

Phase 07 已实现一条自建 HITL 审批链路：Agent 通过 `ToolGuard` 的 `approval_callback` 创建审批请求，后端用 `GET /approval/stream` 推送 SSE，前端 `ApprovalBridge`/`ApprovalDialog` 展示审批，再通过 `POST /approval/{request_id}` 回传决策。该路径可作为回退，但它绕开了 AG-UI 的标准 interrupt/resume 语义。

本地代码已经完成 official adapter spike：`apps/agent/pyproject.toml` 升级为 `pydantic-ai[ag-ui]>=2.3.0` 与 `pydantic-deep>=0.3.34`；`apps/agent/src/server/main.py` 删除自建 resume 提取与 `DeferredToolResults` 转换，让 `/copilotkit` 将原始 body 交给 `AGUIAdapter.dispatch_request()`；`apps/agent/src/agent/paladin_agent.py` 在 `agui_interrupt` 模式下使用 `interrupt_on` 和 `Tool(..., requires_approval=True)`；`apps/desktop/src/components/approval/AguiApprovalInterrupt.tsx` 使用 CopilotKit `useInterrupt(renderInChat: true)` 处理官方 `tool_call` interrupt，并以 `{ approved: true/false }` resume。

Phase 07.1 的作用是把这条 official AG-UI 路径固化为 GSD phase 交付，而不是扩大范围去删除 Phase 07 的 legacy SSE fallback。legacy cleanup 应作为后续 Phase 07.2 单独处理。

## Requirements

1. **Official AG-UI dependency baseline**: Agent dependency set must include official AG-UI support from full `pydantic-ai[ag-ui]` and a Pydantic Deep version that exposes deferred approval integration.
   - Current: Phase 07 used `pydantic-ai-slim[ag-ui]` and a lower `pydantic-deep` version while Paladin carried a local bridge for approval mapping.
   - Target: `apps/agent/pyproject.toml` requires `pydantic-ai[ag-ui]>=2.3.0` and `pydantic-deep>=0.3.34`; `uv.lock` resolves these packages.
   - Acceptance: Dependency probe test imports `AGUIAdapter`, `DeferredToolRequests`, `DeferredToolResults`, `ToolApproved`, and `ToolDenied`, and confirms `RunAgentInput.resume` plus official interrupt/resume protocol types are available.

2. **No custom approval bridge in the AG-UI path**: The custom `agui_approval_bridge.py` mapping layer must not remain in the default AG-UI approval path.
   - Current: Phase 07 spike had local conversion helpers that mapped Pydantic deferred approvals to AG-UI interrupts and resume entries back to `DeferredToolResults`.
   - Target: The local bridge module and its bridge-specific tests are removed; `/copilotkit` relies on official `AGUIAdapter` behavior instead of importing Paladin bridge helpers.
   - Acceptance: `apps/agent/src/agent/agui_approval_bridge.py` and `apps/agent/tests/test_agui_approval_bridge.py` do not exist, and server tests assert `main` imports current `AGUIAdapter` rather than deprecated `handle_ag_ui_request`.

3. **Server preserves official RunAgentInput resume**: `/copilotkit` must pass the complete incoming request body, including `resume`, to `AGUIAdapter.dispatch_request()`.
   - Current: The server manually extracted `resume` entries and passed a locally constructed `deferred_tool_results` argument to `AGUIAdapter.dispatch_request()`.
   - Target: The endpoint replays the original JSON body into a Starlette `Request` and calls `AGUIAdapter.dispatch_request(request=..., agent=..., deps=...)` without local resume conversion.
   - Acceptance: Server tests post a body containing `resume: [{ interruptId, status, payload: { approved: true } }]` and verify the adapter receives the body unchanged.

4. **Agent uses Pydantic deferred approval in `agui_interrupt` mode**: Configured approval-required tools must be routed through Pydantic Deep interrupt semantics when `hitl.mode` is `agui_interrupt`.
   - Current: Phase 07 used `ToolGuard.require_approval` plus `approval_callback` for the self-managed SSE approval path.
   - Target: `agui_interrupt` mode maps `hitl.require_approval` tool names into `interrupt_on`, clears `ToolGuard.require_approval`, and does not install an `approval_callback`.
   - Acceptance: Agent tests verify `create_deep_agent()` receives `interrupt_on == {"write_file": true, "execute": true}` for configured tools, while the guard has `require_approval == []` and `approval_callback is None`.

5. **Computer Use remains approval-gated through native tool approval**: Computer Use tools must still require approval under the official AG-UI path.
   - Current: Computer Use tools were appended to `require_approval` and enforced through `ToolGuard` in the legacy path.
   - Target: In `agui_interrupt` mode, Computer Use functions are registered as `pydantic_ai.Tool` instances with `requires_approval=True`.
   - Acceptance: Agent tests patch Computer Use tool creation and verify registered tools are `Tool` instances with `requires_approval is True`.

6. **Frontend renders official tool-call interrupts in chat**: The desktop approval card must render inside the CopilotKit message stream only for official AG-UI tool-call interrupts.
   - Current: The first implementation recognized Paladin-specific `reason: "approval_required"` and used a local resume payload shape.
   - Target: `AguiApprovalInterrupt` uses `useInterrupt({ renderInChat: true })`, accepts only events whose value has `reason: "tool_call"`, and renders `ApprovalCard` in the chat stream.
   - Acceptance: Frontend tests verify `isApprovalRequired({ value: { reason: "tool_call" } })` returns true; non-`tool_call` interrupts are not approval interrupts by this predicate.

7. **Frontend uses official resume payload schema**: User approval decisions must resolve with the official Pydantic AI AG-UI payload shape.
   - Current: The frontend resolved approval actions with `{ decision: "approved" }` or `{ decision: "denied", message: ... }`.
   - Target: Approve resolves `{ approved: true }`; deny resolves `{ approved: false, reason: "User denied this tool call." }`.
   - Acceptance: Frontend tests verify `buildApprovalResumePayload("approved")` and `buildApprovalResumePayload("denied")` exactly match the target payloads.

## Boundaries

**In scope:**
- Upgrade Agent dependencies to official `pydantic-ai[ag-ui]>=2.3.0` and `pydantic-deep>=0.3.34`.
- Remove Paladin's local AG-UI approval bridge from the default approval path.
- Delegate `/copilotkit` interrupt/resume handling to official `AGUIAdapter`.
- Route `agui_interrupt` approval-required tools through Pydantic Deep `interrupt_on`.
- Register Computer Use tools with native `Tool(..., requires_approval=True)` in `agui_interrupt` mode.
- Render official `tool_call` interrupts as approval cards in the CopilotKit chat stream.
- Use official `{ approved: boolean, reason?: string }` resume payloads.
- Preserve targeted backend and frontend tests for the official path.

**Out of scope:**
- Removing Phase 07 `legacy_sse` fallback — cleanup should be planned as Phase 07.2 after official AG-UI approval is verified.
- Deprecating `legacy_sse` — this phase keeps it as an available fallback without changing its lifecycle status.
- Rewriting `ApprovalBridge`, `ApprovalDialog`, `approval_store.py`, or `/approval/stream` — these belong to the legacy fallback path.
- Changing Computer Use behavior beyond approval routing — Phase 07 already owns screenshot/click/type behavior.
- Adding audit logs or approval history persistence — Phase 09 Admin Systems owns audit concerns.
- Fixing unrelated frontend `Titlebar` unused-variable build errors — they are known build blockers outside the approval path.

## Constraints

- `agui_interrupt` remains the default HITL mode.
- `legacy_sse` must remain selectable via `hitl.mode` for fallback during this phase.
- The `/copilotkit` endpoint must keep using Pydantic AI `AGUIAdapter.dispatch_request()` as the AG-UI entrypoint.
- The frontend approval card must render through CopilotKit `useInterrupt` with `renderInChat: true`.
- This phase may rely on official Pydantic AI protocol behavior instead of duplicating resume parsing in Paladin code.
- Full frontend build may remain blocked only by pre-existing, unrelated `Titlebar` unused-variable errors; approval-specific tests must still pass.

## Acceptance Criteria

- [ ] `apps/agent/pyproject.toml` requires `pydantic-ai[ag-ui]>=2.3.0` and `pydantic-deep>=0.3.34`.
- [ ] `apps/agent/src/agent/agui_approval_bridge.py` and `apps/agent/tests/test_agui_approval_bridge.py` are removed.
- [ ] `/copilotkit` calls `AGUIAdapter.dispatch_request(request=..., agent=..., deps=...)` without manually extracting `resume` or building `DeferredToolResults`.
- [ ] A `/copilotkit` request containing `resume` is forwarded to the official adapter with the body unchanged.
- [ ] In `agui_interrupt` mode, configured `require_approval` tools are passed as `interrupt_on` and `ToolGuard.approval_callback` is not installed.
- [ ] Computer Use tools are registered as `Tool(..., requires_approval=True)` in `agui_interrupt` mode.
- [ ] The desktop approval interrupt renderer accepts `reason: "tool_call"` and does not treat non-`tool_call` interrupts as approval interrupts.
- [ ] Approve resolves `{ approved: true }`; deny resolves `{ approved: false, reason: "User denied this tool call." }`.
- [ ] Backend AG-UI approval tests pass.
- [ ] Frontend approval interrupt tests pass.
- [ ] `npm run build` failures are allowed only when limited to pre-existing `Titlebar` unused-variable errors unrelated to 07.1.
- [ ] `legacy_sse` remains available as a fallback and is not removed or deprecated in this phase.
