# Phase 07.1 — UI Design Contract

> Visual and interaction contract for official AG-UI deferred tool approval interrupts rendered inline inside CopilotKit chat.

---

## Scope

Phase 07.1 adds no broad approval redesign. The official approval UI path is `AguiApprovalInterrupt` using `useInterrupt({ renderInChat: true })`, gated only to official AG-UI interrupts whose `reason` is exactly `"tool_call"`, and rendered through the existing `ApprovalCard` visual language.

Locked behavior:

- Approve resolves exactly `{ approved: true }`.
- Deny resolves exactly `{ approved: false, reason: "User denied this tool call." }`.
- `ApprovalBridge` and `ApprovalDialog` remain legacy fallback UI and must not be removed, redesigned, or made the official AG-UI path in this phase.
- The interrupt predicate must not broaden beyond `reason: "tool_call"`.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn initialized |
| Preset | base-nova |

| Icon library | lucide |
| Font | Geist Variable |
| Source | `apps/desktop/components.json`, `apps/desktop/src/index.css`, `apps/desktop/src/components/ui/*` |

Implementation contract:

- Prefer existing local components and tokens: `Button`, `AlertDialog` only if a confirmation is later required, `border`, `card`, `muted`, `foreground`, `destructive`, `ring`.
- This phase does not require a new shadcn registry install.
- Do not introduce third-party registry blocks, new UI kits, or bespoke global tokens.
- `ApprovalCard` may be refined, but it must remain reusable by both official and legacy paths unless a future cleanup phase explicitly separates them.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Interrupt mount | `AguiApprovalInterrupt` remains mounted inside `ChatArea` adjacent to `CopilotChat`, so CopilotKit can render the card inline in the chat stream. |
| Eligibility | Render only when `event.value` is an object and `event.value.reason === "tool_call"`. Missing, empty, `"approval_required"`, or any other reason is not an approval interrupt. |
| Approval | Clicking `Approve` immediately sets card status to approved, disables both actions, and calls `resolve({ approved: true })`. |
| Denial | Clicking `Deny` immediately sets card status to denied, disables both actions, and calls `resolve({ approved: false, reason: "User denied this tool call." })`. |
| Confirmation | No extra destructive confirmation dialog for denial in 07.1. Denial is reversible only by sending a new agent request, because the official resume payload is final for the current interrupt. |
| Double-click protection | After either action, both buttons are disabled and visually muted. Repeated clicks must not send a second resolve. |
| Expired interrupt | If `expiresAt` is present and elapsed, show status `Expired`, disable actions, and keep the card readable in the transcript. If runtime expiry handling is unavailable, preserve the existing status model without inventing polling. |
| Error | If resolving fails or render data is malformed enough to prevent action, show `Approval error`, disable actions, and leave tool name fallback as `Tool call`. |
| Loading | No spinner required for default pending state. The status pill `Waiting for approval` is the loading/waiting indicator. |
| Empty data | Missing metadata, tool args, message, or tool id must still render a compact card titled `Tool call`; omit empty sections rather than showing placeholders. |
| Legacy fallback | `ApprovalBridge`/`ApprovalDialog` may still appear only through `legacy_sse` mode. Official AG-UI interrupts must not open the legacy dialog. |

## Visual Contract

Focal point: the approval card is the active decision object inside the CopilotKit transcript. It should read as part of the conversation, not as a modal or global alert.

Hierarchy:

1. Status pill: current approval state.
2. Tool name: monospace label, strongest content text in the card.
3. Arguments summary: muted code block, capped/truncated, break long paths and JSON.
4. Message/reason: secondary explanatory text.
5. Actions: `Deny` then `Approve`, aligned in the card footer.

Card boundaries:

- Use `bg-card`, `border-border`, `shadow-sm`, and compact padding.
- Keep card width `w-full` within CopilotKit chat content; do not float, overlay, portal, or pin it outside the chat transcript.
- Keep radius at the existing `rounded-lg` token from the local system. Do not introduce oversized rounded cards.
- Avoid nested cards. The args summary may use a muted inset block but not another card shell.

Icons and accessibility:

- Icons are optional for 07.1. If added, use lucide only: `ShieldCheck` or `Check` for approve, `Ban` or `X` for deny, `AlertTriangle` for error/expired.
- Icon-only controls are not allowed for approve/deny in this phase; text labels must remain visible.
- Buttons must be native buttons or local `Button` components, keep disabled semantics, and expose accessible names exactly matching the visible labels.
- Focus states must use existing `ring`/`focus-visible` token behavior if migrating to local `Button`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Pending status | `Waiting for approval` |
| Approved status | `Approved` |
| Denied status | `Denied` |
| Expired status | `Expired` |
| Error status | `Approval error` |
| Primary CTA | `Approve` |
| Deny CTA | `Deny` |
| Default title | `Tool call` |
| Default prompt | `Approve this tool call?` when the interrupt does not provide a message and the component needs visible body copy. |
| Empty args | Omit the args block; do not display `undefined`, `null`, `{}`, or `No arguments`. |
| Empty tool name | Use `Tool call`. |
| Error state | `Approval error` status with actions disabled; keep displayed tool metadata for diagnosis. |
| Expired state | `Expired` status with actions disabled; do not send a denial payload automatically. |
| Denial reason payload | `User denied this tool call.` |
| Destructive confirmation | None in 07.1; denial is a single explicit `Deny` action. |

Tone:

- Use short, concrete English labels for protocol-facing approval UI, matching current tests and payload behavior.
- Do not add explanatory in-app text about AG-UI, CopilotKit, `useInterrupt`, resume payloads, or implementation mechanics.

## Color

Use the existing neutral shadcn token system. The intended visual split is:

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `background`, `foreground` | Chat canvas and normal readable text. |
| Secondary (30%) | `card`, `muted`, `border`, `muted-foreground` | Approval card shell, args summary, secondary metadata, boundaries. |
| Accent (10%) | status-specific accent only | Status pill and approve affordance. |
| Destructive | `destructive` | Deny button, denial/destructive focus/error affordances only. |

Accent reserved for:

- Pending status pill, currently amber.
- Approved state or approve CTA, currently emerald.
- Focus/ring feedback from the design system.

Accent must not be used for all interactive elements, large backgrounds, decorative gradients, or unrelated chat chrome. Destructive color is reserved for denial/destructive affordances and approval errors; do not use destructive for neutral pending state.

---

## Typography

Use no more than four sizes and two weights inside the approval UI.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Code / pill | 12px (`text-xs`) | 500 | 16px | Status pill, args summary, compact metadata. |
| Body | 14px (`text-sm`) | 400 | 20px | Message and reason text. |
| Tool label | 14px (`text-sm`) | 600 | 20px | Tool name, monospace. |
| Dialog fallback only | 18px (`text-lg`) | 600 | 28px | Existing legacy `ApprovalDialog` title if shown in fallback mode. |

Constraints:

- Do not add hero-scale typography.
- Keep letter spacing at default `0`.
- Long tool names, file paths, and JSON args must wrap with `break-all` or equivalent to avoid overflow.

---

## Spacing Scale

Declared values must remain on the standard 4/8/16/24/32/48/64 scale:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/text gaps, compact inline spacing. |
| sm | 8px | Button gap, pill row gap, card vertical rhythm. |
| md | 16px | Default approval card padding. |
| lg | 24px | Legacy dialog padding only; not needed for inline card expansion. |
| xl | 32px | Empty chat/welcome layout outside this phase. |
| 2xl | 48px | Not used in approval card. |
| 3xl | 64px | Not used in approval card. |

Current acceptable sub-scale exceptions:

- `2px` status pill vertical padding (`py-0.5`) is acceptable because it preserves compact inline badge height.
- `6px` args block vertical padding (`py-1.5`) is acceptable as an existing Tailwind half-step for code readability.

No new spacing exceptions should be introduced in 07.1.

## State Matrix

| State | Visual | Actions | Payload |
|-------|--------|---------|---------|
| Pending | Amber status pill, normal card border, both buttons enabled. | `Deny`, `Approve` enabled. | None until user acts. |
| Approved | Approved status pill, actions disabled with opacity. | Disabled. | `{ approved: true }` already sent. |
| Denied | Denied status pill, actions disabled with opacity. | Disabled. | `{ approved: false, reason: "User denied this tool call." }` already sent. |
| Expired | Expired status pill, actions disabled. | Disabled. | None automatically. |
| Error | Approval error status, destructive/error styling limited to status/action affordance. | Disabled. | None automatically. |
