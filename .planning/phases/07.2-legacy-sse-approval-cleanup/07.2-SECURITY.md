# Phase 07.2 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| config file -> agent factory | Local config selects approval mode before tools are registered. | HITL approval mode and tool approval policy |
| agent factory -> Pydantic Deep | Approval policy crosses into the official deferred approval runtime. | `interrupt_on` approval requirements and native Tool approval flags |
| desktop/client -> FastAPI server | Client requests can reach registered HTTP routes. | AG-UI request/resume payloads |
| FastAPI server -> AG-UI adapter | Official approval resume payloads are delegated to the Pydantic AI adapter. | `/copilotkit` request body and `resume[]` entries |
| desktop UI -> agent server | Legacy EventSource/HTTP decision channel previously crossed this boundary outside official AG-UI. | Removed legacy approval stream and decision traffic |
| CopilotKit AG-UI client -> official interrupt component | Official interrupts are rendered and resumed through CopilotKit/AG-UI client state. | Approval card state and official resume calls |
| docs -> developer/operator behavior | Active docs can cause a developer to configure or rely on unsupported approval paths. | Developer guidance and architecture instructions |
| active source/tests -> release confidence | Residual legacy symbols in active code/tests can preserve unsupported behavior. | Source gates and regression evidence |
| official AG-UI path -> approval-required tools | Cleanup must not weaken approval enforcement. | Approval-required tool calls and resume payloads |
