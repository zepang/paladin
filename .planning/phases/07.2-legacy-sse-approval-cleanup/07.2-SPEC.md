# Phase 07.2: Legacy SSE Approval Cleanup — Specification

**Created:** 2026-07-03
**Ambiguity score:** 0.12 (gate: ≤ 0.20)
**Requirements:** 6 locked

## Goal

Remove Paladin's retained legacy SSE/HTTP HITL approval fallback so the official Pydantic AI AG-UI interrupt/resume path is the only supported approval path.

## Background

Phase 07 implemented a self-managed HITL approval path: `ToolGuard.approval_callback` created `request_id` approval state, `GET /approval/stream` pushed approval requests over SSE, `POST /approval/{request_id}` returned the user's decision, and the desktop used `ApprovalBridge` plus `ApprovalDialog` to show the legacy approval UI.

Phase 07.1 made the official AG-UI approval path the default and verified it through backend, frontend, and manual UAT. The official path now uses Pydantic Deep deferred approvals, AG-UI `RUN_FINISHED.outcome.interrupts`, the desktop `AguiApprovalInterrupt` card, and AG-UI `resume[]` entries. The old legacy SSE fallback was intentionally retained in 07.1 and documented in `07.1-07.2-HANDOFF.md` for this cleanup phase.

Today, legacy code still exists in `apps/agent/src/agent/hitl.py`, `apps/agent/src/agent/approval_store.py`, `apps/agent/src/server/main.py`, `apps/desktop/src/components/approval/ApprovalBridge.tsx`, `apps/desktop/src/components/approval/ApprovalDialog.tsx`, `apps/desktop/src/App.tsx`, `apps/desktop/src/components/ChatToolbar.tsx`, and legacy-only tests. This phase removes that fallback and verifies that official AG-UI approval remains intact.

## Requirements

1. **Remove legacy mode selection**: Paladin must no longer support `hitl.mode = "legacy_sse"`.
   - Current: `_approval_mode()` accepts both `"agui_interrupt"` and `"legacy_sse"`, and `create_paladin_agent()` installs an `approval_callback` when legacy mode is selected.
   - Target: `agui_interrupt` is the only supported approval mode; explicit `"legacy_sse"` config is rejected with a clear configuration error.
   - Acceptance: Agent tests verify default mode remains `agui_interrupt`, `legacy_sse` raises `ValueError`, and no branch installs `approval_callback` for legacy fallback.

2. **Remove backend legacy approval endpoints**: The FastAPI server must not expose the old SSE/HTTP approval endpoints or their broadcast infrastructure.
   - Current: `apps/agent/src/server/main.py` imports `hitl`, bridges `hitl._sse_queue`, starts/stops a broadcast task, keeps `_approval_queues`, and defines `GET /approval/stream` plus `POST /approval/{request_id}`.
   - Target: Those routes, queues, broadcast lifecycle handlers, and server-level HITL bridge imports are removed.
   - Acceptance: Server tests verify `/approval/stream` and `/approval/{request_id}` are not registered, and source checks find no server references to `_approval_queues`, `_broadcast_approval_event`, `approval_stream`, or `approval_decision`.

3. **Remove legacy approval state and callback code**: Request-id based approval callback/store code must not remain as an active implementation path.
   - Current: `hitl.py` owns `_pending_approvals`, `_pending_decisions`, `_sse_queue`, `create_approval_callback()`, `get_pending_approval()`, and `resolve_approval()`; `approval_store.py` contains legacy approval record persistence not used by the official AG-UI path.
   - Target: Legacy callback/store code and its tests are removed or reduced to non-approval configuration validation only; no active module maintains request-id approval state.
   - Acceptance: Legacy-only tests are deleted or rewritten, and source checks find no active references to `create_approval_callback`, `_pending_approvals`, `_pending_decisions`, `_sse_queue`, `resolve_approval`, or `approval_store` in `apps/agent/src` and active tests.

4. **Remove desktop legacy approval bridge UI**: The desktop must not create an EventSource to `/approval/stream`, post decisions to `/approval/{request_id}`, or render the old approval dialog/status path.
   - Current: `ApprovalBridge.tsx` opens an EventSource and posts decisions; `ApprovalDialog.tsx` consumes that context; `App.tsx` wraps the app in `ApprovalProvider`; `ChatToolbar.tsx` reads pending legacy approvals.
   - Target: Legacy bridge/dialog/provider/status wiring is removed; official approval cards continue to render through `AguiApprovalInterrupt`.
   - Acceptance: Frontend tests and source checks verify `ApprovalBridge`, `ApprovalDialog`, `ApprovalProvider`, `useApprovalContext`, `/approval/stream`, and `/approval/{request_id}` are absent from active desktop source while `AguiApprovalInterrupt` remains mounted in the chat area.

5. **Clean legacy-only tests and stale active documentation**: Tests and active phase docs must not present legacy SSE approval as available fallback behavior after this phase.
   - Current: `test_hitl.py`, `test_approval_store.py`, and several phase artifacts describe the SSE fallback as retained or selectable.
   - Target: Legacy-only tests are removed or replaced with official-path regression checks; active ROADMAP/STATE/phase docs describe 07.2 as removing the fallback. Historical artifacts may retain chronology only when clearly marked as prior-phase history.
   - Acceptance: Repository search for `legacy_sse`, `/approval/stream`, `/approval/{request_id}`, `ApprovalBridge`, `ApprovalDialog`, and `approval_store` returns no active source/test references except historical planning docs that explicitly describe prior behavior or this cleanup record.

6. **Preserve official AG-UI approval behavior**: Official approval must remain functional after legacy cleanup.
   - Current: Phase 07.1 verified `agui_interrupt`, `interrupt_on`, native `Tool(..., requires_approval=True)` Computer Use tools, `AguiApprovalInterrupt`, and AG-UI `resume[]` payloads.
   - Target: The same official approval behavior remains after deleting the fallback.
   - Acceptance: Targeted backend agent/server tests and frontend approval interrupt tests pass, including configured approval-required tool interruption, Computer Use approval gating, approval card rendering for `reason: "tool_call"`, and official `resume[]` payload submission.

## Boundaries

**In scope:**
- Remove `legacy_sse` as a supported HITL mode.
- Remove backend legacy SSE/HTTP approval routes and broadcast infrastructure.
- Remove legacy request-id approval callback/store implementation and legacy-only tests.
- Remove desktop legacy approval bridge, dialog, provider, and toolbar status wiring.
- Update active docs so legacy SSE is no longer described as an available fallback.
- Preserve and regression-test the official AG-UI approval path.

**Out of scope:**
- Redesigning `ApprovalCard` visuals — 07.2 is cleanup, not a UI redesign.
- Changing which tools require approval — existing `hitl.require_approval` and Computer Use approval semantics remain.
- Reintroducing Paladin-side AG-UI resume conversion — Phase 07.1 already removed that bridge.
- Adding audit log persistence or approval history — Phase 09 owns audit/admin concerns.
- Implementing Tauri sidecar management — deferred Phase 7b/Phase 8 work.
- Fixing unrelated `Titlebar` TypeScript build errors — they are not approval cleanup.

## Constraints

- `agui_interrupt` remains the default and only supported approval mode.
- Explicit `hitl.mode = "legacy_sse"` must not silently fall back to official approval.
- `/copilotkit` must continue delegating to Pydantic AI `AGUIAdapter.dispatch_request()`.
- `hitl.require_approval`, `hitl.blocked`, and `timeout_seconds` configuration semantics remain available unless they are proven legacy-only during planning.
- Computer Use approval remains mandatory through the official AG-UI path.
- Historical `.planning` artifacts may mention legacy SSE as prior behavior, but active source/tests must not depend on it.
