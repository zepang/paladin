<objective>
用 TDD 实现一个**纯函数**的状态机模块：`ProcessEvent` 枚举（SpawnStarted/ProbeOk/ProbeFail/ProbeDegraded/Crashed/SpawnFailed/RestartRequested/BackoffTick/StopRequested）+ `transition(state, event, snapshot) -> (new_state, new_snapshot)` + `backoff_delay(attempt_idx, max_restarts) -> Option<Duration>`。所有逻辑零 I/O，毫秒级执行，覆盖 SPEC Acceptance 中关于状态转换、3 连败阈值、启动失败 vs 运行崩溃区分、退避序列、退避耗尽的所有硬约束。

Purpose: supervisor (plan 06) 的复杂度集中在并发 + I/O + Tauri 集成层；把「状态决策」剥离为纯函数后，可以用密集单元测试覆盖每条 SPEC 边界，避免在集成层才发现逻辑错误。这是 TDD 启发式「Business logic with defined I/O → type: tdd」的典型适用。
Output: 一份 `cargo test -p paladin-desktop --lib process::state_machine` 全绿、覆盖率包含 5×6 状态-事件矩阵的纯 Rust 模块。
</objective>

<feature>
  <name>ProcessState 状态机 + 退避序列 (R2, R3, R4)</name>
  <files>apps/desktop/src-tauri/src/process/state_machine.rs</files>
  <behavior>
    RED cases (all must fail before implementation):
    - 初始状态：`transition(Starting, ProbeOk, &default)` → `(Running, snap_with_fail_count_0)`。
    - 3 连败阈值：连续 3 次 `transition(_, ProbeFail, _)` 后状态变为 Unhealthy；2 次 ProbeFail 后仍 Running（fail_count=2 < 3）。
    - ProbeDegraded 不累加 fail_count：`transition(Running, ProbeDegraded, _)` → `(Degraded, snap_with_fail_count_unchanged)`；从 Degraded 收到 ProbeOk → Running。
    - 启动失败 vs 运行崩溃：`transition(Starting, SpawnFailed{4.9}, _)` → `(Stopped, _)`（4.9 < 5.0 走启动失败路径，**不**进入 backoff）；`transition(Running, Crashed{10.0}, _)` → `(Unhealthy, snap_with_restart_count_incremented)`（10.0 > 5.0 走 backoff 路径）。
    - 5s 边界含：`transition(Starting, SpawnFailed{5.0}, _)` → `(Stopped, _)`（边界值 5.0 也归启动失败）。
    - 退避序列：`backoff_delay(0, 5) == Some(1000ms)`、`backoff_delay(1, 5) == Some(2000ms)`、`backoff_delay(2, 5) == Some(4000ms)`、`backoff_delay(3, 5) == Some(8000ms)`、`backoff_delay(4, 5) == Some(16000ms)`、`backoff_delay(5, 5) == None`（耗尽）。
    - 手动重启：`transition(Stopped, RestartRequested, _)` → `(Starting, snap_with_restart_count_0_and_fail_count_0)`；任何状态收到 RestartRequested 都重置 fail_count=0、restart_count=0、state=Starting。
    - Stopped 不自恢复：`transition(Stopped, BackoffTick{attempt: 100}, _)` → `(Stopped, _)`（不变，禁止后台唤醒）。
    - StopRequested 在任何状态都 → Stopped。
  </behavior>
  <implementation>transition 用 match (current, event) 决策。Crashed 与 SpawnFailed 的区分点：event 自带 exited_after_secs 字段，<= STARTUP_GRACE_SECS 视为 SpawnFailed（即使事件是 Crashed 也按启动失败处理 — 但本特征用两个独立 event 表达，由 supervisor 层在 spawn 后 5s 内 exit 转 SpawnFailed）。backoff_delay 用 attempt_idx 作为 BACKOFF_SEQUENCE_MS 索引；attempt_idx >= max_restarts 或 attempt_idx >= BACKOFF_SEQUENCE_MS.len() 返回 None。SupervisorSnapshot 是 Copy + Clone，transition 返回新 snapshot 而非 mutate。</implementation>
</feature>
