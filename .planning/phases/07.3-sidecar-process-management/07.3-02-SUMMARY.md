## transition() 状态-事件决策表

`transition` 的核心 match 矩阵(完整文档见 [state_machine.rs](file:///Users/kdocs/Workspace/paladin/apps/desktop/src-tauri/src/process/state_machine.rs)):

| 当前状态 \ 事件 | ProbeOk | ProbeFail | ProbeDegraded | SpawnFailed | Crashed | RestartRequested | BackoffTick | StopRequested |
|-----------------|---------|-----------|---------------|-------------|---------|------------------|-------------|---------------|
| **Starting** | Running (fc=0) | 累加 fc,达 3 → Unhealthy | Degraded | **Stopped** (不消费 backoff) | Unhealthy (rc+1) | Starting (清零) | Starting | Stopped |
| **Running** | Running (fc=0) | 累加 fc,达 3 → Unhealthy | Degraded | Stopped | Unhealthy (rc+1) | Starting (清零) | Running | Stopped |
| **Degraded** | Running (fc=0) | 累加 fc,达 3 → Unhealthy | Degraded | Stopped | Unhealthy (rc+1) | Starting (清零) | Degraded | Stopped |
| **Unhealthy** | Running (fc=0) | 累加 fc | Degraded | Stopped | Unhealthy (rc+1) | Starting (清零) | attempt ≥ 5 → Stopped, 否则 Unhealthy | Stopped |
| **Stopped** (终态) | **Stopped** (P3) | **Stopped** (P3) | **Stopped** (P3) | Stopped | Stopped | **Starting (清零)** (唯一脱离路径) | Stopped (P3) | Stopped |

`fc` = fail_count,`rc` = restart_count。

## Decisions Made

- **transition 不二次判定 `exited_after_secs`**:`SpawnFailed` / `Crashed` 的语义由 supervisor 层在 spawn 后 5s 内/外 exit 自动分类(PLAN implementation 明确说明)。state machine 只按 event 类型路由,避免边界判定逻辑分散两处。
- **`SupervisorSnapshot` 全 `u32` 字段**:`last_error_len` 替代 `Option<String>` 以支持 `Copy + Eq`,transition 返回新 snapshot 而非 mutate — supervisor 跨线程传递无并发顾虑,测试断言也简洁(`assert_eq!(snap1, snap2)` 可直接用)。
- **Stopped 终态硬特殊处理**:match 开头先判定 `current == Stopped`,只接受 RestartRequested 脱离。这条 P3 硬约束如果散落在每个事件分支容易漏,集中处理更安全。
- **`#![allow(dead_code)]` module-level 抑制**:state_machine 模块的 pub 项在 plan 06 supervisor 集成前仅被测试调用,与 plan 01 supervisor.rs 的 `#[allow(dead_code)] pub struct ProcessSupervisor` 同款"骨架先行"模式 — 集成后 warning 自然消失。
