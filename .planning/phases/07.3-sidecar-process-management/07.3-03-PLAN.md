<objective>
用 TDD 实现 processes.json 加载器与 schema 校验。Plan 01 已定义了 serde 结构体（ProcessConfig/ProcessEntry/HealthConfig/EndpointConfig/ProcessNameKey），本 plan 补 `pub fn load_from_path(path: impl AsRef<Path>) -> Result<ProcessConfig, ConfigError>` 与 `pub enum ConfigError { NotFound, IoError(io::Error), ParseError(serde_json::Error), InvalidSchema(String) }`，并写覆盖正反两面的测试。

Purpose: 配置加载是 SPEC Edge R1 的关键防线（配置错 → 不 spawn、报配置错误）。把加载/校验抽离为纯函数 + 测试覆盖，能让 plan 06 supervisor 在调用 load_from_path 时对错误处理零模糊。
Output: `cargo test -p paladin-desktop --lib process::config` 全绿；plan 01 的 processes.json 能被 round-trip 解析。
</objective>

  <behavior>
    RED cases (all must fail before implementation):
    - 合法 round-trip：用 `include_str!("../../processes.json")` 或临时文件加载 plan 01 写的 processes.json，断言 cfg.mode == "dev"、cfg.processes 含 agent 与 server、agent.cmd == ["uv","run","paladin-agent","serve","--dev"]、server.port == 9880、backoff_secs == [1,2,4,8,16]、max_restarts == 5。
    - 文件不存在：load_from_path("/nonexistent/path/processes.json") 返回 Err(ConfigError::NotFound)。
    - JSON 损坏：临时文件写入 `{` 不完整 JSON，返回 Err(ParseError)。
    - cmd 单字符串：临时文件写入 `{"mode":"dev","processes":{"agent":{"cmd":"uv run foo","port":9876,"health":{"liveness":{"path":"/health","expect_status":200}},"startup_grace_secs":5}},"backoff_secs":[1,2,4,8,16],"max_restarts":5,"shutdown_grace_secs":5}`，serde 拒绝（Vec<String> 类型不匹配 string），返回 Err(ParseError)。
    - env 字面值：临时文件 env 字段为 `{"FOO": "${BAR}"}`，反序列化后 cfg.processes.agent.env["FOO"] == "${BAR}"（字面值）。
    - port 越界：port=0 或 port=65536，validate() 返回 Err(InvalidSchema)。
    - cmd 空数组：cmd=[]，validate() 返回 Err(InvalidSchema)。
    - 缺 liveness：agent entry 缺 health.liveness 字段，serde 拒绝（liveness 非Option），返回 Err(ParseError)。
    - backoff_secs 长度异常：backoff_secs=[1,2,3,4,5,6]（>5），validate() 返回 Err(InvalidSchema)。
    - max_restarts 越界：max_restarts=99，validate() 返回 Err(InvalidSchema)。
  </behavior>
  <implementation>load_from_path 三步：exists() 检查 → fs::read → serde_json::from_slice → validate()。validate() 检查：每个 process 的 cmd 非空 Vec、port 在 1..=65535、health.liveness.expect_status 在 100..=599、readiness（若存在）的 expect_status 同范围；顶层 backoff_secs.len() <= 5、max_restarts <= 5、shutdown_grace_secs > 0 && <= 60。ConfigError 用 thiserror（若未引入则手写）。env 字段直接保留 serde 解析的字面 HashMap，不做任何 shell 展开或路径解析（${VAR} 是字面字符串）。</implementation>
</feature>
