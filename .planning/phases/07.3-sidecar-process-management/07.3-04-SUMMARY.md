## Decisions Made
- **采用 PATTERNS §6.1 途径 A（手写）而非途径 B（shadcn CLI）**：与 alert-dialog.tsx 同源保证视觉/交互一致，避免联网拉取 + 可能的 schema 不一致
- **PopoverContent 用 Portal + Positioner(sideOffset=4) + Popup 三层组合**：浮层必须锚定 trigger（区别于 alert-dialog 的 fixed-centered 居中弹窗）
- **Trigger as-child prop 名 = `render`**：核实自 `node_modules/@base-ui/react/internals/types.d.ts` 的 `BaseUIComponentProps.render?: React.ReactElement | ComponentRenderFn`，与 PATTERNS §6.2 一致；plan 08 ProcessLight 应使用 `render={<Button .../>}` 而非 radix 风格的 `asChild`
- **不引入 React.ComponentProps<'div'> 辅助导出**：alert-dialog 有 Header/Footer/Title/Description 等布局槽位，Popover 在 v1 只暴露 Root/Trigger/Content 三件，布局槽位延后到消费方按需添加（YAGNI）
- **删除计划中字面指定的 `import type * as React from 'react'`**：本组件未直接引用 React 命名空间类型（无 React.ComponentProps 用法），保留会导致 TS6133 未用导入错误（Task 1 自动修复）
