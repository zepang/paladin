<objective>
用 TDD 实现两个纯函数模块，为 plan 06 supervisor 提供「spawn 前端口预检」与「日志落盘/推送前脱敏」两个已验证的决策内核：

1. **port_probe** — `is_port_available(port: u16) -> bool`：基于 std::net::TcpListener::bind(("127.0.0.1", port)) 同步探测，bind 成功 = 空闲（drop listener），bind 失败 = 占用。覆盖 SPEC R1 端口冲突报错 + Edge R9 孤儿进程检测。

2. **log_redact** — `redact_log_line(line: &str) -> String`：对 DEEPSEEK_API_KEY / JWT_SECRET / PALADIN_JWT_SECRET / Authorization Bearer / password 五类 secret 模式做 case-insensitive 掩码，值替换为 `[REDACTED]`。继承 Phase 8 D-15「日志绝不含敏感信息」约束。

两个模块零 tokio / reqwest / std::process，毫秒级测试，为 plan 06 supervisor 的并发 + I/O 层提供正确性已验证的纯函数依赖。

Purpose: 把「端口冲突判定」与「日志脱敏」从 supervisor 集成层剥离，避免在并发 + Tauri Channel + Tauri State 复杂层才发现边界 bug。
Output: `cargo test --manifest-path apps/desktop/src-tauri/Cargo.toml process::port_probe process::log_redact` 全绿。
</objective>
