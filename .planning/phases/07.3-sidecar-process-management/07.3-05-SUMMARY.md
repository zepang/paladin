## port_probe 算法

```
is_port_available(port: u16) -> bool:
    TcpListener::bind(("127.0.0.1", port))
        .is_ok()   // Ok → 空闲 (listener 局部变量,函数返回时 drop)
                   // Err → 占用 / 权限不足 / OS 拒绝
```

关键不变量:函数返回 `bool` 不持有 listener,因此 listener 在函数返回时立即 drop — `test_listener_dropped_after_check` 显式证明 (连续两次 `is_port_available(p)` 同一空闲 p 都返回 true)。

`check_port_or_error(port, process_name)` 是 supervisor-friendly 包装:
```rust
if is_port_available(port) { Ok(()) }
else { Err(format!("端口 {} 已被占用({} 进程无法启动),请释放后重试", port, process_name)) }
```

## log_redact 5 类正则模式

| # | secret 类别 | 正则模式 (case-insensitive) | 替换模板 | 覆盖测试 |
|---|---|---|---|---|
| 1 | DEEPSEEK_API_KEY | `(?i)(DEEPSEEK_API_KEY\s*[=:]\s*)\S+` | `${1}[REDACTED]` | test_redact_deepseek_api_key + test_redact_case_insensitive (三形态) |
| 2 | JWT_SECRET | `(?i)(JWT_SECRET\s*[=:]\s*)\S+` | `${1}[REDACTED]` | test_redact_jwt_secret + test_redact_placeholder_not_leaking_original |
| 3 | PALADIN_JWT_SECRET | `(?i)(PALADIN_JWT_SECRET\s*[=:]\s*)\S+` | `${1}[REDACTED]` | test_redact_paladin_jwt_secret |
| 4 | Authorization Bearer | `(?i)(Authorization\s*:\s*Bearer\s+)\S+` | `${1}[REDACTED]` | test_redact_authorization_bearer |
| 5 | password (含 JSON) | `(?i)(password\s*["']?\s*[=:]\s*["']?)\S+` | `${1}[REDACTED]` | test_redact_password + test_redact_password_json_form |

5 个 `LazyLock<Regex>` 在首次调用 `redact_log_line` 时一次性编译,后续毫秒级匹配。

password 模式设计要点:`["']?` 可选地匹配 JSON 形态的引号,`[=:]` 同时覆盖 `=` / `:` 分隔符,`\s*` 容忍空白。`\S+` 贪婪匹配值直到空白或行尾,保证 `hunter2"}` 这种 JSON 值末尾的非空白字符也被一并替换。

## Decisions Made

- **`is_port_available` 用 bool 表达而非自定义 Result/enum**:supervisor 集成层只需 yes/no 决策;若需要错误细节,用 `check_port_or_error` 包装。两个 API 共存,职责单一。
- **`check_port_or_error` 错误消息中文**:与项目 STATUS BAR / Toast / 启动遮罩 (D-06) 文案风格一致,plan 06 supervisor `spawn_one()` 直接 `?` 传播,前端遮罩直接显示。
- **log_redact 用 regex crate 而非手写 split/replace**:5 类模式 + case-insensitive + JSON 形态 + key 名保留 + 分隔符保留,手写状态机会很复杂且易漏。regex 一行一个模式,可读性高,编译后毫秒级。
- **占位符用字面 `[REDACTED]` (含方括号)**:与 k8s secret / docker log redaction 等行业惯例一致;前端日志面板可直接渲染,无需 escape。`test_redact_placeholder_not_leaking_original` 显式证明占位符不可被反向构造为原值。
- **password 模式用 `r#"..."#` raw string**:模式内含双引号字符 (`["']?`),普通 raw string `r"..."` 无法包含 `"`,必须用 hash raw string `r#"..."#`。这是 Rust 字符串语法限制,与正则本身无关。
- **模块层 `#![allow(dead_code)]`**:与 plan 02 state_machine.rs / plan 03 config.rs 同款"骨架先行"模式。port_probe / log_redact 在 plan 06 supervisor 集成前仅被测试调用,集成后 warning 自然消失。
