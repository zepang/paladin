## SPEC Prohibitions 在代码层面的防范

- **P1(命令注入面收敛)**:`spawn_child_to_slot` 内 `Command::new(&entry.cmd[0]).args(&entry.cmd[1..])`,cmd 来自 ProcessConfig 静态字段,tokio::process::Command 默认不走 shell。
- **P2(last_error + stderr_tail 双暴露)**:`emit_status_to` 在每次 transition 后序列化 `ProcessStatusPayload { name, state, last_error, stderr_tail, last_restart_at }`,stderr_tail 由 capture_lines 维护末尾 50 行 VecDeque。
- **P3(stopped 不自恢复)**:wait_exit SpawnFailed(≤5s)→ Stopped 直接 break;backoff_delay None(max_restarts 耗尽)→ 手动转 Stopped;probe_loop 看到 Stopped 退出;无后台 timer 唤醒 Stopped 进程。
- **P4(仅 tracked Child handle 操作)**:graceful_shutdown_slot 从 `slot.child.lock().await.take()` 取 child handle,基于 child.id() 发信号;绝不读 PID 文件、不系统 kill 匹配。
- **Edge R10(logs-dir-not-writable 降级)**:`tokio::fs::OpenOptions::open` 失败 → file=None → loop 继续 emit;后续写失败也降级为 None 避免反复失败占用 CPU。

## Tauri 事件契约(前端 plan 07 消费)

### `process-status` payload(ProcessStatusPayload)
```jsonc
{
  "name": "agent" | "server",
  "state": "starting" | "running" | "degraded" | "unhealthy" | "stopped",
  "last_error": string | null,         // 中文人类可读退出原因
  "stderr_tail": string | null,         // 末尾 50 行 stderr 用 \n 连接
  "last_restart_at": number | null      // Unix epoch 毫秒,手动 restart 时刷新
}
```
触发时机:每次 transition 导致 state 变化、Starting 状态、Running 清错、Stopped 终态。

### `process-log` payload(LogChunk)
```jsonc
{
  "process": "agent" | "server",
  "stream": "stdout" | "stderr",
  "line": string,        // 已过 redact_log_line 脱敏
  "ts": number           // Unix epoch 毫秒
}
```
触发时机:子进程每输出一行(stream 区分),前端按 (process, stream) 路由到对应 console tab。

### `get_runtime_config` 返回值(RuntimeConfig)
```jsonc
{
  "agent_url": "http://localhost:9876",
  "server_url": "http://localhost:9880"
}
```
URL 端口来自 processes.json,D-07 单一 Rust 真相源。

### `get_process_status` 返回值(ProcessStatusSnapshot)
```jsonc
{
  "agent":  { "state": "...", "last_error": ..., "stderr_tail": ..., "last_restart_at": ... },
  "server": { "state": "...", "last_error": ..., "stderr_tail": ..., "last_restart_at": ... }
}
```
