<objective>
把前端的「健康检查 + URL 配置 + 启动遮罩」三件事从「前端自发 fetch + 硬编码 URL + 二态降级」迁移到「Rust emit 订阅 + runtimeConfig 驱动 + 5 状态遮罩」。本 plan 是 plan 06 supervisor 的前端消费层，为 plan 08（StatusBar 多灯）与 plan 09（LogsPanel）提供 store 基础。

具体改动：
1. 新建 `stores/process.ts` — zustand 单 store，initProcessListeners() 全局订阅 'process-status'。
2. 改造 `useAgentHealth.ts` — 删除 fetch，改 selector 订阅 store.agent，保留 isOnline/isLoading/error/retry 接口名。
3. 新建 `StartupMask.tsx` — starting(蓝 spinner)/unhealthy(黄)/stopped(红) + last_error + stderr_tail + restart 按钮（SPEC Prohibition P2 UI 侧闭环）。
4. 改造 `App.tsx` — invoke('get_runtime_config') → 替换 3 处 URL → 遮罩从二态升 5 状态 → StatusBar 始终渲染 → CopilotKitProvider 仅 running 时 mount。
5. `main.tsx` 调用 initProcessListeners()。

Purpose: SPEC R2（前端订阅）+ R8（URL 配置驱动）+ D-05/D-06/D-07/D-08/D-10 全部在前端层闭环。
Output: grep localhost:9876 业务代码 0 命中；useAgentHealth 零 fetch；agent 不 healthy 时遮罩常驻。
</objective>

## Artifacts this phase produces

**stores/process.ts：**
- `export type ProcessName = 'agent' | 'server';`
- `export type ProcessState = 'starting' | 'running' | 'degraded' | 'unhealthy' | 'stopped';`
- `export interface ProcessInfo { state: ProcessState; last_error: string | null; stderr_tail: string | null; last_restart_at: number | null; }`
- `interface ProcessStore { agent: ProcessInfo; server: ProcessInfo; setStatus: (name: ProcessName, patch: Partial<ProcessInfo>) => void; }`
- `export const useProcessStore = create<ProcessStore>(...)`
- `export const useProcessStatus = (name: ProcessName) => useProcessStore((s) => s[name]);`
- `export async function initProcessListeners(): Promise<() => void>` — invoke('get_process_status') 拉初值 + listen('process-status') 订阅后续，返回 unlisten。

**hooks/useAgentHealth.ts：**
- 保留 `interface AgentHealthState { isOnline, isLoading, error }` 与返回 `{ ...state, retry }`。
- 内部实现切到 `useProcessStatus('agent')` selector + `invoke('restart_agent')` 作为 retry。
- `isOnline = state === 'running'`；`isLoading = state === 'starting'`；`error = last_error ?? (state !== 'running' ? fallback 文案 : null)`。

**components/StartupMask.tsx：**
- `export function StartupMask({ agentState, error, stderrTail, onRestart }: { agentState: ProcessState; error: string | null; stderrTail: string | null; onRestart: () => void; })`
- starting → 蓝 spinner + "正在启动 Agent..."；unhealthy → 黄 + "Agent 异常，正在自动重启"；stopped → 红 + "Agent 已停止" + last_error + stderr_tail `<pre>` + 重启按钮。

**App.tsx 改动：**
- 顶部加 `const [runtimeConfig, setRuntimeConfig] = useState<RuntimeConfig | null>(null);` + useEffect invoke('get_runtime_config') + catch fallback `{ agentUrl: 'http://localhost:9876', serverUrl: 'http://localhost:9880' }` + console.warn。
- `const agentStatus = useProcessStatus('agent');` + `const agentState = agentStatus.state;`
- `const agUiAgent = useMemo(() => runtimeConfig && new HttpAgent({ url: \`${runtimeConfig.agentUrl}/copilotkit\` }), [runtimeConfig]);`
- CopilotKitProvider runtimeUrl 改 `${runtimeConfig!.agentUrl}/copilotkit`（仅 runtimeConfig && agentState==='running' 时渲染）。
- render 顺序：runtimeConfig==null → spinner；agentState !== 'running' → Titlebar + StartupMask + StatusBar + Toaster；else → CopilotKitProvider 包主界面。

**main.tsx 改动：**
- 顶层 useEffect 或 init 调用 `initProcessListeners()`（返回 unlisten 在 cleanup 调用）。
