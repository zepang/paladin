## Decisions Made

1. **base-ui `render={<Button/>}` 而非 radix `asChild`** — plan 04 SUMMARY 已核实 base-ui Trigger 的 as-child prop 名是 `render`(非 radix 的 `asChild`),源 自 `node_modules/@base-ui/react/internals/types.d.ts` `BaseUIComponentProps.render?: React.ReactElement | ComponentRenderFn`。本 plan 是该契约的首个真实消费方,grep `asChild` 在 ProcessLight.tsx 0 命中,grep `render=` 1 命中,严格对齐。
2. **5 状态色值映射不走 token 体系** — `bg-emerald-500`/`bg-amber-500`/`bg-orange-500` 是 raw Tailwind 色,与终端灯的 `bg-green-500` 视觉协调;`bg-primary`/`bg-destructive` 走 design token。**不**强迁全 token 体系,因为那意味着同时改终端灯 raw 色,超出本 plan scope(PATTERNS §5.2 "一致性优先 — 继承现有终端灯 raw 色")。
3. **省略 `<PopoverContent align="start">` 的 align prop** — base-ui 中 `align` 是 `PopoverPositioner.Props` 的字段(verified via `PopoverPositioner.d.ts: align: Align`),不是 `PopoverPopup.Props`;plan 04 锁定的 PopoverContent 内部固化 `<Positioner sideOffset={4}>` 不透传 align。最小 scope 解是省略 align,默认锚定足够(本 plan 不修 plan 04 的 popover.tsx,避免跨 plan 文件 scope)。详见 Deviations。
4. **terminal.ts 类型扩展前置** — plan 09 PATTERNS §7.2 F5 本就要做 `activePanel: 'terminal' | 'file-preview' | 'diff' | 'logs'`,本 plan 因调用 `setActivePanel('logs')` 必须先把类型联合扩好。最小变更:仅扩 2 处类型签名(`activePanel` 字段 + `setActivePanel` 入参),不改初始值/不改 RightPanel tabs(那些仍是 plan 09 工作)。详见 Deviations。

ProcessLight.tsx 中的 trigger 写法(plan 04 SUMMARY 锁定的 base-ui 契约):

```tsx
<PopoverTrigger
  render={
    <Button variant="ghost" size="sm" title={`${label}: ${text}`}>
      <span className={`inline-block w-2 h-2 rounded-full ${DOT_CLASS[status.state]}`} />
      <span className="text-xs">
        {label}
        {status.state === 'running' ? '' : `·${text}`}
      </span>
    </Button>
  }
/>
```

### 5 状态色值/文案映射

| ProcessState | DOT_CLASS | TEXT | 中文 |
|---|---|---|---|
| starting | bg-primary(蓝) | '启动中' | 首探完成前 / spawn 后窗口内 |
| running | bg-emerald-500(绿) | '运行中' | 健康探针 200 |
| degraded | bg-amber-500(黄) | '降级' | Go /readyz 非 200(PG/Redis 挂) |
| unhealthy | bg-orange-500(橙) | '异常' | 探针 3 连败,重启中 |
| stopped | bg-destructive(红) | '已停止' | 启动失败 / 退避耗尽 / 手动 stop |
