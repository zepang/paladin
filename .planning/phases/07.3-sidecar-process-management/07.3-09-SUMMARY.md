## Decisions Made

1. **LogChunk 字段命名严格对齐 Rust serde** — Rust `LogChunk` struct(`supervisor.rs:90-101`)+ `#[serde(rename_all = "lowercase")]` 在 ProcessName/LogStream 上,实测 JSON payload = `{ process: "agent"|"server", stream: "stdout"|"stderr", line: string, ts: number }`。前端 TypeScript `interface LogChunk` 严格匹配这一形态,避免字段读不到 undefined。
2. **Task 1 no-op 处理** — plan 08 SUMMARY §Deviations 1 已记录 `terminal.ts activePanel` 类型扩展是 plan 09 工作的前置(因 plan 08 的 ProcessLight 调用 `setActivePanel('logs')` 必须先有类型)。本 plan Task 1 acceptance(grep + tsc)全部 PASS,无代码改动,不创建空提交。
3. **暂停语义选择** — PATTERNS §7.3 提供两种暂停范式("drop" vs "append 不滚动");本 plan 严格遵循 PLAN `<action>` 字面的 `if (paused) return`(drop 当前行)。v1 简单优先,未来 phase 若需"暂停期间不丢日志"可改为 ringbuffer + 恢复时回填。
4. **stderr 前缀用 `⚠ ` 而非 ANSI 着色** — LogsPanel 渲染在 `<div className="whitespace-pre-wrap">` 而非 xterm 终端,ANSI 色码不会被解析;`⚠ `(U+26A0 + 空格)在 pre-wrap 白背景下视觉清晰且不依赖终端色码。
5. **buffer 容量 5000 行 + shift()** — v1 已知限制:无界追加会内存爆炸,5000 行足够定位近期问题;日志轮转/大小上限/压缩 deferred 到 Phase 10 ops(plan 06 已在文件层处理落盘轮转,前端 buffer 仅影响显示窗口)。

## LogsPanel 与 process store logs API 的对齐证明

LogsPanel 不消费 `stores/process.ts` 的 `useProcessStatus` selector(那是 plan 07 的 process-status 事件消费层,供 StartupMask/ProcessLight 用);LogsPanel 直接 `listen('process-log')` 消费 plan 06 supervisor emit 的 LogChunk。对齐证明如下:

| 前端 LogsPanel 消费 | plan 06 Rust emit 字段(supervisor.rs:90-101 LogChunk) | serde 序列化 | 对齐 |
|---|---|---|---|
| `e.payload.process: 'agent' \| 'server'` | `process: ProcessName` (Agent/Server enum) | `#[serde(rename_all = "lowercase")]` → `"agent"` / `"server"` | ✓ lowercase 匹配 LogTab type |
| `e.payload.stream: 'stdout' \| 'stderr'` | `stream: LogStream` (Stdout/Stderr enum) | `#[serde(rename_all = "lowercase")]` → `"stdout"` / `"stderr"` | ✓ lowercase 匹配 |
| `e.payload.line: string` | `line: String` (已 redact_log_line 脱敏) | 默认 snake_case | ✓ 字段名一致 |
| `e.payload.ts: number` | `ts: i64` (now_ms Unix epoch 毫秒) | i64 → JSON number | ✓ number 类型匹配 |
| `listen<LogChunk>('process-log', ...)` | `app.emit("process-log", &chunk)` (supervisor.rs:970) | event name 字面匹配 | ✓ 事件名一致 |
| 按 `payload.process` 路由到 `buffersRef.current[process]` | emit 时 `process: name`(ProcessName) | — | ✓ agent→agent buffer, server→server buffer |
| stderr 行加 `⚠ ` 前缀 | `stream == LogStream::Stderr` 时 emit | — | ✓ 前端按 `stream === 'stderr'` 加前缀 |

**安全约束闭环:** LogsPanel 不含 `invoke('get_runtime_config')` 或读取 `processes.json` env 字段(grep 验证 0 命中),仅消费 `process-log` event 的 `line` 字段(plan 05 + plan 06 双重脱敏)。`grep -cE "get_runtime_config\|processes.json\|env\b" LogsPanel.tsx` → 0。
