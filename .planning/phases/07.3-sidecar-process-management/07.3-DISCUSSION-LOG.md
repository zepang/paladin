## 启动时序与首屏体验

| Option | Description | Selected |
|--------|-------------|----------|
| 等 ready 再 mount | Agent healthy 才 mount CopilotKitProvider，首屏遮罩。CopilotKit 是 fetch-per-request 无重试。 | ✓ |
| 立即 mount + 降级占位 | UI 显示"启动中"占位，发消息走等待。体验不一致。 | |
| 立即 mount + 消息队列 | 发消息走队列稳态后重放。需自建队列，复杂度高。 | |

**User's choice:** 等 ready 再 mount
**Notes:** CopilotKit 无持久连接健康概念，不能让它自己重试。

| Option | Description | Selected |
|--------|-------------|----------|
| 状态驱动遮罩 | starting 蓝/unhealthy 黄/stopped 红 + 退出原因 + restart。继承 SPEC P2。 | ✓ |
| 超时后进主界面 | 30s 超时消失，状态栏报错。错误感知滞后。 | |
| 遮罩仅 starting | 失败跳主界面。错误信息不集中。 | |

**User's choice:** 状态驱动遮罩
**Notes:** 首屏即错误屏。

| Option | Description | Selected |
|--------|-------------|----------|
| Rust 推送 get_runtime_config | Rust 唯一真相源，暴露命令返回 {agentUrl, serverUrl}。dev/packaged 统一。 | ✓ |
| dev .env + packaged fetch | dev 用 VITE_AGENT_URL，packaged fetch config.json。分支逻辑。 | |
| 混合 | dev import.meta.env，packaged Tauri 命令。两套路径。 | |

**User's choice:** Rust 推送 get_runtime_config
**Notes:** SPEC R8 import.meta.env 路径简化为单一 Rust 源（前端业务代码仍 0 命中 9876）。

| Option | Description | Selected |
|--------|-------------|----------|
| 状态驱动重新出现 | 手动 stop/unhealthy/stopped 后遮罩重临。与启动对称。 | ✓ |
| 仅启动遮罩 | stop 后主界面保留 + 发消息禁用。不对称。 | |

**User's choice:** 状态驱动重新出现
**Notes:** Agent 不 healthy 时遮罩常驻。
