### 2.4 子进程 spawn + Tauri Channel 日志推送

[terminal/mod.rs:111-155](file:///Users/kdocs/Workspace/paladin/apps/desktop/src-tauri/src/terminal/mod.rs#L111-L155) 是项目内唯一的 "spawn 子进程 + 把 stdout 流推 Tauri Channel" 范本：

```rust
let _child = pair.slave.spawn_command(cmd)
    .map_err(|e| format!("启动 shell 失败: {}", e))?;
// ...
let shutdown_flag = Arc::new(AtomicBool::new(false));
let flag_clone = shutdown_flag.clone();
let id_clone = id.clone();
let app_clone = self.app_handle.clone();

std::thread::spawn(move || {
    let mut buf = [0u8; 4096];
    loop {
        if flag_clone.load(Ordering::Relaxed) { break; }
        match reader.read(&mut buf) {
            Ok(0) => { let _ = app_clone.emit("pty-eof", &id_clone); break; }
            Ok(n) => { if channel.send(buf[..n].to_vec()).is_err() { break; } }
            Err(_) => { let _ = app_clone.emit("pty-error", &id_clone); break; }
        }
    }
});
```

**关键复用点：**

1. `Channel<Vec<u8>>` 作为 Tauri command 入参（前端 invoke 时序列化）——SPEC R6 日志 Channel 直接套用，但 payload 改为结构化 `LogChunk { process: ProcessName, stream: Stdout|Stderr, bytes: Vec<u8>, ts: i64 }`，便于前端按 tab 分流。
2. `Arc<AtomicBool>` 旗标 + 独立线程读循环——监督器 probe/log 任务用 `tokio::spawn` + `tokio::select!` 替代（SPEC 已要求 `tokio::process`，D-14 信任 planner 选具体并发模型）。
3. `app.emit("pty-eof", &id)` 模式——supervisor 改 emit `"process-status"`，payload = `{ name, state, last_error?, last_stderr_tail? }`（满足 SPEC Prohibition P2 "不静默重启，必须暴露退出原因"）。

> ⚠️ TerminalManager 用 `portable-pty` 是因为 PTY 交互；supervisor **不要**沿用 `portable-pty`，改用 `tokio::process::Command`（D-13 锁定）。

### 2.9 跨平台终止机制（SPEC R5 / R9）

SPEC Constraints "Windows 无 SIGTERM" + Prohibition P4 "MUST NOT 凭裸 PID 跨会话 kill"。三条硬约束：

1. **必须**用 `tokio::process::Child` 的 `kill()` —— 它内部走 Job Object（Windows）/ SIGKILL（Unix），且只对当前会话 spawn 的 child handle 有效（满足 P4）。
2. **优雅关闭** = 先 `child.start_kill()` 不行的话（实际上 tokio 没有 SIGTERM API），planner 需在 SPEC 硬约束内自决：
   - **Unix**：用 `nix` crate 发 `SIGTERM` → 等 5s → `child.kill()` 兜底；或 `libc::kill(pid, SIGTERM)`。
   - **Windows**：`taskkill /PID <pid> /T`（递归子进程）/ Job Object 关闭 / `TerminateProcess`。D-14 信任 planner 选其一或组合。
3. **不**写 PID 文件、**不**跨会话匹配 PID（P4 验证：子进程死后 PID 复用，关闭时**不**杀哨兵）。

```rust
// 推荐骨架（planner 自决具体 crate）
async fn graceful_shutdown(child: &mut tokio::process::Child) {
    #[cfg(unix)]
    { let _ = nix::sys::signal::kill(
        nix::unistd::Pid::from_raw(child.id() as i32),
        nix::sys::signal::Signal::SIGTERM); }
    #[cfg(windows)]
    { /* taskkill /T or Job Object */ }

    if tokio::time::timeout(Duration::from_secs(5), child.wait()).await.is_err() {
        let _ = child.kill().await;   // 兜底
    }
}
```

```ts
      .then((snap) => setState(snap.agent.state))
      .catch(() => setState('stopped'));

    const unlisten = listen<ProcessStatusPayload>('process-status', (event) => {
      if (event.payload.name !== 'agent') return;
      setState(event.payload.state);
      setError(event.payload.last_error ?? null);
    });
    return () => { void unlisten.then((fn) => fn()); };
  }, []);

  const retry = useCallback(() => { invoke('restart_agent').catch(() => {}); }, []);
  return { state, isOnline: state === 'running', isLoading: state === 'starting', error, retry };
}
```

## 8. 进程状态 store（F10）

参照 [stores/terminal.ts:31-77](file:///Users/kdocs/Workspace/paladin/apps/desktop/src/stores/terminal.ts#L31-L77) 的 zustand 范式。**单一 store** 集中订阅 Rust event，避免多个组件各自 listen：

```ts
import { create } from 'zustand';
import { listen } from '@tauri-apps/api/event';
import { invoke } from '@tauri-apps/api/core';

export type ProcessName = 'agent' | 'server';
export type ProcessState = 'starting' | 'running' | 'degraded' | 'unhealthy' | 'stopped';

interface ProcessInfo {
  state: ProcessState;
  last_error: string | null;
  stderr_tail: string | null;
  last_restart_at: number | null;
}

interface ProcessStore {
  agent: ProcessInfo;
  server: ProcessInfo;
  setStatus: (name: ProcessName, patch: Partial<ProcessInfo>) => void;
}

const initial: ProcessInfo = {
  state: 'starting', last_error: null, stderr_tail: null, last_restart_at: null,
};

export const useProcessStore = create<ProcessStore>((set) => ({
  agent: { ...initial },
  server: { ...initial },
  setStatus: (name, patch) => set((s) => ({ [name]: { ...s[name], ...patch } })),
}));

// 全局启动监听一次（在 main.tsx 或 App.tsx 顶层 useEffect 调用）
export async function initProcessListeners() {
  // 拉一次初值
  const snap = await invoke<Record<ProcessName, ProcessInfo>>('get_process_status');
  useProcessStore.setState({ agent: snap.agent, server: snap.server });

  // 订阅后续更新
  const unlisten = await listen<{ name: ProcessName } & Partial<ProcessInfo>>(
    'process-status',
    (e) => {
      const { name, ...patch } = e.payload;
      useProcessStore.getState().setStatus(name, patch);
    }
  );
  return unlisten;
}
```

`useProcessStatus(name)` selector（[ProcessLight.tsx](file:///Users/kdocs/Workspace/paladin/apps/desktop/src/components/StatusBar) 用）：


**Rust serde 草案：**

```rust
#[derive(Deserialize)]
pub struct ProcessConfig {
    pub mode: String,
    pub processes: HashMap<ProcessNameKey, ProcessEntry>,
    pub backoff_secs: Vec<u64>,
    pub max_restarts: u32,
    pub shutdown_grace_secs: u64,
}

#[derive(Deserialize, Hash, Eq, PartialEq)]
#[serde(rename_all = "lowercase")]
pub enum ProcessNameKey { Agent, Server }

#[derive(Deserialize)]
pub struct ProcessEntry {
    pub cmd: Vec<String>,    // SPEC Prohibition P1: 必须数组
    pub cwd: Option<PathBuf>,
    pub env: HashMap<String, String>,
    pub port: u16,
    pub health: HealthConfig,
    pub startup_grace_secs: u64,
}

#[derive(Deserialize)]
pub struct HealthConfig {
    pub liveness: EndpointConfig,
    pub readiness: Option<EndpointConfig>,
}

#[derive(Deserialize)]
pub struct EndpointConfig {
    pub path: String,
    pub expect_status: u16,
}
```

`ProcessNameKey` 可 `impl From<ProcessNameKey> for ProcessName`，把配置层与运行时 enum 解耦。
