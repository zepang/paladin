# 07.4-01 Plan: Runtime Tuple Model And Packaged Validation Foundation

<objective>
建立 Rust 侧可测试的 runtime tuple 基础：`ProcessState + ProcessOwner + ProcessHealth`、`conflict`、合法组合表、mode-specific config validation、packaged no-dev-deps 约束。后续 supervisor 与 frontend 只能消费这层已验证的语义，不在 async/runtime/UI 代码里重新解释状态。
</objective>

<execution_context>

</execution_context>

<threat_model>
- Threat: packaged app accidentally executes developer tools from user PATH. Mitigation: packaged validation rejects `uv`/`go`; later spawn path must avoid `augmented_spawn_path`.
- Threat: invalid status tuples hide ownership safety violations. Mitigation: pure legality table rejects impossible combinations.
- Threat: config failure gets buried in stderr. Mitigation: validation returns stable diagnostic strings consumed by later user-visible config diagnostic work.
- Threat: command injection via config refactor. Mitigation: preserve `cmd: Vec<String>` and shell-free execution; do not add single-string parsing.
</threat_model>
