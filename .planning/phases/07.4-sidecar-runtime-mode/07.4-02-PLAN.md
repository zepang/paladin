# 07.4-02 Plan: Rust Supervisor Hybrid Runtime

<objective>
把 07.4-01 的 tuple/model 基础接入 Rust supervisor：`get_process_status` 与 `process-status` 发出 `state + owner + health`，dev startup/redetect 先探测健康再决定 attach/spawn/conflict，packaged spawn 保持 supervisor-owned 且不依赖 dev PATH，external stop/restart/shutdown/recovery 在 Rust 边界被拒绝或跳过。
</objective>

This plan depends on 07.4-01. It changes Rust runtime behavior and must use behavior tests where feasible. Standard execute task type is used because it wires async supervisor and Tauri commands, but every code-changing task below has `tdd="true"` and a behavior block.
</execution_context>

<threat_model>
- Threat: external process killed accidentally. Mitigation: Rust owner guards in stop/restart/shutdown and no PID/port kill behavior.
- Threat: dev healthy service misreported as conflict. Mitigation: liveness/readiness probe before port bind.
- Threat: occupied malicious/unrelated service treated as Paladin. Mitigation: expected HTTP status is the locked attach criterion; failed configured health becomes conflict, not attach.
- Threat: double-spawn from concurrent redetect/restart. Mitigation: per-service runtime lock covers all lifecycle actions.
- Threat: packaged build executing dev tools. Mitigation: config validation from 07.4-01 plus packaged spawn branch without PATH augmentation.
</threat_model>
