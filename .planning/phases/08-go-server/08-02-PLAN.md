<objective>
Provide the local PostgreSQL + Redis dependencies and the versioned auth schema that every downstream data/auth plan depends on. Bring up `docker-compose.server.yml`, install the golang-migrate CLI, author the `users`/`roles`/`user_roles` migrations with UNIQUE constraints (including a case-insensitive email normalization index), and prove idempotency by running `migrate up` twice and exercising every duplicate-rejection constraint.

Purpose: Plans 08-03 (sqlc), 08-05 (auth), and 08-07 (integration) all require a live PG with a stable, constraint-backed schema. This plan delivers that contract and is parallel-safe with 08-01 (no Go needed: the migrate CLI is installed via Homebrew and the schema is pure SQL).
Output: A compose stack reachable on 5432/6379, a migration set that round-trips cleanly up/down, and verified duplicate-rejection at the DB layer.
</objective>
