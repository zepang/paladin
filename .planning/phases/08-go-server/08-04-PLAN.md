<objective>
Construct the Gin HTTP server: the unified error writer, panic recovery middleware, the `/healthz` liveness and `/readyz` PG+Redis probes, the sample `/me` and `/admin/health` handler stubs (auth-wired in 08-05), and the `NewServer` composition function. Wire `cmd/server/main.go` to build config -> pool -> redis -> engine -> `http.Server` with graceful shutdown on SIGTERM/SIGINT.

Purpose: This plan delivers the SRV-01 server skeleton and the health endpoints that the SPEC R1 acceptance criteria require, plus the error-shape contract that every later handler depends on. The auth handlers (08-05) and WebSocket route (08-06) plug into the engine and middleware established here.
Output: A `go run ./cmd/server` process that binds :9880, serves `/healthz` (200) and `/readyz` (200 when compose is up, 503 when a dependency is down), and returns unified errors on any non-2xx path.
</objective>
