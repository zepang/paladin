<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client credentials -> register/login | Untrusted email/password reach bcrypt + sqlc before a JWT is minted. |
| JWT -> protected routes | A forged or replayed token must not grant access. |
| role claims -> RBAC | Attacker-controlled role ordering must not change the decision. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-05-01 | Spoofing | JWT alg confusion | high | mitigate | WithValidMethods([\"HS256\"]) + SigningMethodHMAC assertion; alg:none test (RESEARCH Security Domain). |
| T-08-05-02 | Information Disclosure | secret echo | high | mitigate | Unified errors + negative tests assert no token/password in bodies or logs (Prohibition 1). |
| T-08-05-03 | Elevation of Privilege | RBAC bypass | high | mitigate | Membership-based RequireRole + order-independent test; user -> 403 enforced (Prohibition 3). |
| T-08-05-04 | Tampering | email identity | high | mitigate | normalizeEmail on insert AND lookup; DB UNIQUE(LOWER(email)) backstop (R3 encoding/concurrency edges). |
| T-08-05-05 | Repudiation | duplicate admin grant | medium | mitigate | UNIQUE(user_id, role_id) + ON CONFLICT DO NOTHING; BootstrapAdmin idempotent (R4 adjacency edge). |
| T-08-05-SC | Tampering | package installs | low | accept | golang-jwt/v5 and golang.org/x/crypto already pinned in 08-01; no new modules. |
</threat_model>
