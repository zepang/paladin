<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw TCP -> WS upgrade | Any client can open the /ws HTTP upgrade before auth. |
| first message -> JWT verify | The first frame is untrusted until auth.Verify succeeds. |
| Hub registry -> broadcast fanout | A write to a dead/slow conn must not abort fanout or panic. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-06-01 | Spoofing | WS auth bypass | high | mitigate | First-message handshake + auth-timeout close; Hub never registers pre-auth (D-17, R5 empty edge). |
| T-08-06-02 | Information Disclosure | token in URL/logs | high | mitigate | No query-string auth (D-18); close reasons omit the token (Prohibition 1). |
| T-08-06-03 | Denial of Service | concurrent-write panic | high | mitigate | coder/websocket concurrent-write safety + -race broadcast test (R5 concurrency edge). |
| T-08-06-04 | Elevation of Privilege | broadcast endpoint | medium | mitigate | Broadcast is internal-only; no HTTP route (D-20, negative test). |
| T-08-06-SC | Tampering | package installs | low | accept | coder/websocket v1.8.15 already pinned in 08-01. |
</threat_model>
