<objective>
Deliver ADM-01 audit log persistence: an append-only `audit_logs` PostgreSQL table with JSONB metadata, sqlc-generated insert query, a Recorder service with per-action metadata allowlists that structurally prevents secret leakage, and hybrid recording entry points (Gin middleware for RBAC denials + explicit calls in auth handlers for login/register semantics).

Purpose: Phase 8 explicitly deferred audit history to Phase 9. No audit table, migration, or recording code path exists today. This plan creates the durable audit foundation that R2 (recording) and R3 (query API, plan 09-04) build on.
Output: A passing `go test ./internal/audit/... ./internal/http/middleware/... -race` suite proving audit records are written for login success/failure, registration, and RBAC denials, with zero secret leakage.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| request data -> audit metadata | Untrusted request fields (IP, user-agent, path) enter JSONB metadata. |
| auth success/failure -> audit row | Login/register outcomes must be recorded exactly once without secret leakage. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-09-01-01 | Information Disclosure | secret in metadata | high | mitigate | Per-action allowlist structurally excludes password/hash/JWT/bearer keys (D-03); negative test asserts no secret substring. |
| T-09-01-02 | Repudiation | missing audit row | medium | mitigate | Hybrid middleware+explicit recording covers RBAC denials (middleware) and business events (handlers); concurrency test proves no loss. |
| T-09-01-03 | Tampering | audit row mutation | high | mitigate | Append-only: no UPDATE/DELETE query or method; down migration is the only DROP. |
| T-09-01-SC | Tampering | SQL injection via metadata | low | accept | sqlc generates parameterized queries; metadata is marshalled JSON, not string-interpolated SQL. |
</threat_model>
