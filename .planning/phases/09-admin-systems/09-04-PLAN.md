    - Filters: user_id filter narrows to that user; action filter narrows to that action; status filter narrows to success/failure; composing all three narrows correctly (adjacency edge).
    - Ordering: items are newest-first by created_at (ordering edge).
    - Fan-out: an audit insert triggers hub.SendToRole("admin", ...) and admin conns receive an "audit.log" envelope; non-admin conns do not.
    - Fan-out: a 429 quota_exceeded triggers hub.SendToRole("admin", ...) and admin conns receive a "quota.exceeded" envelope.
    - Secret safety: no audit query response field or fan-out envelope payload contains a password/JWT substring.
  </behavior>

Wire Recorder.SendToRole after insert (goroutine, non-blocking, hub may be nil for tests). Wire QuotaGate.SendToRole on 429 (goroutine). Mount route with Auth + RequireRole("admin"). Envelope payloads carry only allowlisted fields (action, status, user_id, ip) — never secrets.</implementation>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin JWT -> audit query | Only admin-role tokens may read audit history. |
| audit insert -> admin WS fan-out | Audit events carry allowlisted metadata only, never secrets. |
| quota 429 -> admin WS fan-out | Quota-exceeded events notify admins without leaking the rejected token. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-09-04-01 | Information Disclosure | non-admin reads audit | high | mitigate | Auth + RequireRole("admin") gate; 401/403 test matrix (SPEC Prohibition). |
| T-09-04-02 | Information Disclosure | secret in fan-out | high | mitigate | Envelope payloads use allowlisted fields only; no-secret-in-envelope test. |
| T-09-04-03 | Tampering | cursor forgery | low | accept | Cursor is opaque base64 of (ts, id); a forged cursor yields an empty or shifted page, not a bypass. |
| T-09-04-04 | Repudiation | fan-out loss | medium | mitigate | Concurrent-fan-out test proves N inserts → N envelopes delivered. |
</threat_model>
