# 10-02 Plan: Bundle Wiring And Packaged Process Config

<objective>
把 Wave 1 产出的 sidecar binaries 与 packaged process config 接入 Tauri bundle：保留 dev `processes.json`，新增或生成 bundle 使用的 packaged config，使用 `externalBin`/resource 风格让安装后的应用可以读取 sidecar 和 config，而不是回落到源码树。
</objective>

<execution_context>
This plan performs packaging-manifest wiring and packaged config definition. The runtime lookup code remains for 10-03, but this plan must make the bundle inputs concrete and testable. TDD applies to packaged-config validation fixtures and resource/bundle presence checks.
</execution_context>

Implementation notes:
- D-07 requires keeping the current dev `processes.json` unchanged for development.
- D-08 and D-09 require a packaged config that preserves `cmd: string[]`, ships in the bundle, and stays compatible with packaged validation.
- Tauri manifest changes must prepare both `.dmg` and `.msi` outputs without implying release readiness before UAT.
</context>

- `processes.packaged.json` or an equivalent generated packaged config resource
