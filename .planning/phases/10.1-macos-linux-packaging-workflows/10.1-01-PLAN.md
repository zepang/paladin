<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI workflow to manifest CLI | Workflow-provided artifact paths and target strings enter Node validation. |
| Build outputs to uploaded metadata | Installer and sidecar filenames, sizes, and hashes become auditable manifest content. |
| Runner environment to manifest | CI environment may contain secrets but manifest must not serialize env values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10.1-01 | Tampering | `createBuildManifest()` target/filename validation | high | mitigate | Validate exact supported target triples, exact sidecar filenames, installer roles, counts, and extensions before hashing. |
| T-10.1-02 | Information Disclosure | `build-manifest.json` | high | mitigate | Emit only explicit commit, runner OS, target, architecture, filename, size, and SHA-256 metadata; test fake secret absence. |
| T-10.1-03 | Denial of Service | Manifest validation on directories/missing paths | medium | mitigate | Reject missing paths and non-regular files through `lstat()` before streaming hashes. |
| T-10.1-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package-manager install task is planned; existing dependencies only. If an executor adds installs, stop for package legitimacy review. |
</threat_model>
