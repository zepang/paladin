<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Desktop -> Agent runtime API | Secret-bearing provider snapshots cross from trusted local desktop into Agent memory. |
| Chat request -> model resolver | User chat request triggers provider resolution and potential upstream model call. |
| Agent health/readiness -> Desktop UI | Readiness fields inform product UI and must not misrepresent process health. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-02-01 | Information Disclosure | provider_routes.py | high | mitigate | Return only masked metadata/readiness from read routes; raw key accepted only in update/validate request body and never logged. |
| T-11-02-02 | Denial of Service | main.py startup | high | mitigate | Remove import-time concrete model creation; no-key startup and concurrent probe tests must pass. |
| T-11-02-03 | Tampering | ProviderRuntime snapshot update | high | mitigate | Version snapshots and capture at request start so concurrent updates cannot mutate in-flight requests. |
| T-11-02-04 | Spoofing | /health readiness fields | medium | mitigate | Health response must expose Agent liveness and AI readiness as distinct keys with tests for unconfigured state. |
| T-11-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package installs are planned; keep implementation on existing FastAPI/Pydantic AI dependencies. |
