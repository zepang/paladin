Implement the Rust/Tauri app-data provider configuration authority.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Env -> app-data bootstrap | Parent process environment can seed local runtime config. |
| React form -> Rust manager | User-entered provider metadata and secrets cross into Tauri command layer. |
| App data -> readback DTO | Persisted local config becomes ordinary frontend state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-03-01 | Tampering | bootstrap.rs | high | mitigate | Seed only clean config and preserve saved-local precedence so env cannot silently switch provider after explicit selection. |
| T-11-03-02 | Tampering | storage.rs | high | mitigate | Serialize saves and use temp-file rename; tests cover overlapping saves and dangling active references. |
| T-11-03-03 | Information Disclosure | types.rs/storage.rs | high | mitigate | Raw keys remain out of ordinary metadata/readback; masked fixed fingerprint only. |
| T-11-03-04 | Denial of Service | storage load | medium | mitigate | Invalid/corrupt app-data JSON must return a recoverable error and not prevent sidecar process liveness; executor should surface unconfigured/invalid state. |
| T-11-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package installs are planned; use existing serde/tokio/tempfile patterns. |
</threat_model>
