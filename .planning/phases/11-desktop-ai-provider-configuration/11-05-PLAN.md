<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| React -> Tauri commands | User-supplied provider data and write-only key values cross to Rust. |
| Tauri -> Agent runtime API | Raw key snapshot crosses to local Agent memory for request-time model use. |
| Command readback -> React store | Masked provider metadata becomes ordinary UI state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-05-01 | Information Disclosure | tauri-commands.ts / mod.rs | high | mitigate | Read commands and TS DTOs expose only `has_api_key` and fingerprint; tests assert no raw key readback. |
| T-11-05-02 | Tampering | save/set-active commands | high | mitigate | Persist first through manager, then refresh Agent; saved explicit selection prevents env override. |
| T-11-05-03 | Denial of Service | refresh_agent_ai_provider | medium | mitigate | Refresh failure returns recoverable invalid/untested state and does not restart or stop sidecars. |
| T-11-05-04 | Spoofing | readiness DTO mapping | medium | mitigate | Use explicit `unconfigured|untested|available|invalid` enum, not process `running/degraded` terms. |
| T-11-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package installs are planned. |
</threat_model>
