## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status-dot gaps, inline control spacing |
| sm | 8px | Compact rows, tab button inner gaps, field helper spacing |
| md | 16px | Form group spacing, empty-state padding, panel content padding |
| lg | 24px | Provider section padding, chat CTA block padding |
| xl | 32px | Empty-state vertical gap, grouped settings bands |
| 2xl | 48px | Centered empty-state max vertical rhythm only |
| 3xl | 64px | Not used in this phase except inherited app-level empty layout |

Exceptions: RightPanel header remains `36px` high (`h-9`), StatusBar remains `24px` high (`h-6`), compact icon controls may use existing shadcn `24px`, `28px`, and `32px` button sizes. Provider form touch/click targets must not be smaller than `28px` inside the desktop panel.

## Typography

Use only these phase text sizes for new provider UI. Do not introduce hero-scale typography; this is a settings/workflow surface inside an existing desktop tool.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px | 400 | 1.4 |
| Body | 14px | 400 | 1.5 |
| Heading | 16px | 600 | 1.2 |
| Display | 24px | 600 | 1.2 |

Usage:
- `12px`: status bar labels, helper text, provider fingerprints, tab labels, validation hints.
- `14px`: form values, provider rows, body copy, popover text.
- `16px`: provider panel section titles and current-provider summary heading.
- `24px`: chat-area missing-provider empty-state heading only.

Source/default: Geist from current app; sizes follow current RightPanel/StatusBar/ChatArea density.

## Color

Use existing CSS variables from `apps/desktop/src/index.css`; do not add a new palette for this phase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / light `oklch(1 0 0)`, dark `oklch(0.145 0 0)` | App background, RightPanel body, ChatArea base |
| Secondary (30%) | `var(--muted)`, `var(--card)`, `var(--sidebar)` | StatusBar, panel headers, provider rows, form grouping surfaces |
| Accent (10%) | `var(--primary)` with `var(--primary-foreground)` | Primary CTA, active provider selection, selected RightPanel tab, focus/ring treatment |
| Destructive | `var(--destructive)` | Delete provider action and destructive confirmation only |

Accent reserved for: `配置 AI provider` CTA, `保存配置` primary button, active provider badge/radio, selected `AI Provider` tab, keyboard focus rings inherited from shadcn. Do not use accent for every clickable row.

Status colors:
- `unconfigured`: muted dot/text (`text-muted-foreground`, `bg-muted-foreground`).
- `untested`: amber dot/text (`bg-amber-500`, `text-amber-600 dark:text-amber-400`).
- `available`: emerald dot/text (`bg-emerald-500`, `text-emerald-600 dark:text-emerald-400`).
- `invalid`: orange warning for failed validation (`bg-orange-500`, `text-orange-600 dark:text-orange-400`); use destructive only when the user is deleting or replacing data.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | 配置 AI provider |
| Empty state heading | 尚未配置 AI provider |
| Empty state body | 配置 DeepSeek、OpenAI 兼容服务或 LM Studio 后即可开始对话。 |
| Error state | 当前 provider 不可用。请检查 base URL、API key 和模型 ID，或切换到其他 provider。 |
| Destructive confirmation | 删除 provider: 删除后将移除该 provider 的本地配置；如果它是当前激活项，Paladin 会回到未配置状态。 |

Additional copy rules:
- Use neutral product language. Do not blame the user and do not describe missing AI configuration as an Agent crash or Go readiness failure.
- Use `provider` consistently in UI labels for this phase. Do not mix with "模型服务商" unless a later localization pass changes all instances together.
- API key input placeholder: `输入新的 API key` for empty/replacement entry. Never prefill raw secrets.
- Masked key metadata: `API key 已配置 · pk_7F3A` style fixed short fingerprint. Do not reveal raw prefix/suffix or key length.
- Save/test separation labels: `保存配置` and `测试连接`. Testing success copy: `连接可用`; testing failure copy: `连接测试失败，请检查配置后重试`.

Source: `11-CONTEXT.md` decisions D-07 through D-13 and `11-SPEC.md` requirements R2, R3, R6, R7.

## Interaction Contract

Provider-not-configured chat state:
- When the Agent is running but AI readiness is `unconfigured`, `invalid`, or equivalent provider-not-configured response is returned, ChatArea shows a centered empty-state block inside the chat surface.
- The block contains one heading, one body line, and one primary `配置 AI provider` button with a lucide settings-style icon.
- Clicking the CTA opens RightPanel, sets active panel to `ai-provider`, and focuses the first missing required field or provider type selector.
- Repeated failed chat attempts update the same state; they must not stack toasts or open duplicate panels.

RightPanel provider view:
- Add one tab to the existing RightPanel tab row: id `ai-provider`, label `AI Provider`, icon from lucide (`BrainCircuit`, `Settings`, or `KeyRound`; choose one and use consistently).
- The provider view is a fixed panel surface, not a full app route and not a modal-first flow.
- Preserve current RightPanel resize/fullscreen/close behavior and default width rules. The provider form must work at the current minimum panel width of `300px`.
- Layout order: current provider summary, provider list, editor form, validation/test result area.
- Provider rows show display name, type, model ID, readiness status, active marker, and masked key metadata. They must not show raw key or variable values.

Save/test behavior:
- `保存配置` persists metadata and secret replacement independently of network validation.
- `测试连接` validates the current form/provider without making the provider active unless the user explicitly saves or selects it.
- Active provider may be `untested` or `invalid`; the UI must show the state and still allow selection/saving.
- Provider switching applies to the next chat request. Do not imply that in-flight chat will switch mid-stream.

StatusBar provider state:
- Add a separate AI provider status entry beside Agent and Go process lights.
- Label format: `AI · 未配置`, `AI · 未测试`, `AI · 可用`, `AI · 无效`.
- Clicking the status entry opens a Popover with provider name/model/status and actions: `打开设置`, `测试连接` when a provider exists.
- This state must be visually and semantically separate from Agent process health and Go readiness.

Secret input behavior:
- API key fields are empty on edit even when a key exists.
- Existing secret state is shown only as presence plus fixed short masked fingerprint.
- Re-entering a key replaces the stored secret on save.
- Clearing/removing an existing key requires an explicit secondary action in the form; deletion of a provider requires destructive confirmation.

Accessibility:
- Every icon-only button must have `aria-label` and `title`.
- The chat CTA and status popover actions must be keyboard reachable.
- Validation errors must be rendered as text next to the relevant field; color alone is insufficient.
- Focus returns to the triggering CTA/status control when the provider panel is closed.
