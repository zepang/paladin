# Phase 12 — UI 设计契约

> 已安装 Paladin 的 Go 可选服务配置、诊断与恢复界面契约。它复用既有右侧面板和状态栏，不新增路由、依赖或独立诊断中心。

---

## 范围与信息架构

本阶段只扩展三处既有界面；`StartupMask` 仍只由 Agent 阻塞状态触发，Go 未配置或 Go readiness 降级绝不能遮挡工作区。

| 位置 | 入口/组件 | 责任 | 不承担的责任 |
|---|---|---|---|
| 右侧面板 | 新增 `Go 服务` 标签（`GoServicePanel`，Lucide `ServerCog` 或 `Database`，二选一并全程一致） | 输入三项运行配置、查看遮罩配置状态、保存、测试/重试、显式环境导入、清除本地配置、查看字段与运行时诊断 | 不显示原始密钥、不编辑 shell 环境、不安装 PostgreSQL/Redis、不展示 AI provider 配置 |
| 底部 StatusBar | `GoServiceLight` 或对现有 `ProcessLight(name="server")` 的受控扩展 | 紧凑地表达 Go 配置来源、sidecar/端口/健康状态，提供打开设置、测试/重试、重新检测、查看日志 | 不将 Go 状态折叠为 Agent 状态或全局启动失败 |
| ChatArea / StartupMask | 仅沿用既有行为 | Agent 已运行时照常进入工作区；AI 未配置仍使用 AI Provider CTA | 不因 Go 未配置显示全屏遮罩、聊天阻断 CTA 或“配置 AI provider”替代文案 |

右侧面板标签顺序：保留 `终端`、`文件`、`Diff`、`日志`、`AI Provider`；将 `Go 服务` 放在 `AI Provider` 后。若宽度不足，继承既有 `hidden sm:inline` 标签压缩，仅保留有 `title` 的图标按钮。不得新增全局导航、独立设置页、向导或 modal-first 配置流程。

来源：`12-CONTEXT.md` D-01--D-04、D-09--D-12；`apps/desktop/src/components/layout/RightPanel.tsx`、`StatusBar.tsx`、`StartupMask.tsx`。

## 状态模型与映射

前端接收一个结构化、无机密 DTO。具体 Rust 字段名可由实现决定，但 UI 必须能独立呈现以下五个维度，不得从错误字符串猜测状态：

| 维度 | 允许值/含义 | UI 规则 |
|---|---|---|
| AI provider | 未配置、未测试、可用、无效 | 由既有 `AiProviderLight` 单独呈现；绝不写入或复用 Go 面板 |
| Go 本地配置 | `unconfigured`、`invalid`、`configured`、`session-override` | 显示完整性、来源和固定遮罩指纹；`session-override` 必须注明“仅本次启动，未保存” |
| Go 字段诊断 | DB URL、Redis URL、JWT secret 各自为缺失/无效/已配置 | 缺失或无效时逐字段显示文字；不能显示字段值、长度、前后缀或可逆掩码 |
| sidecar/端口 | starting、running、degraded、unhealthy、stopped、conflict，以及 owner/health | 复用 `ProcessLight` 语义；端口冲突与 sidecar 启动失败独立于配置错误 |
| readiness | unknown、checking、ready、dependency-unavailable、failed | `ready` 代表 Go 依赖可用；liveness 成功而 readiness 失败必须是非阻塞 `降级` |

显示优先级从具体到概括：字段缺失/无效 > sidecar 失败/端口冲突 > readiness 依赖不可用 > 可用。即使有多个问题，状态栏只显示最高优先级短标签；Popover 和面板按上述分类逐项列出。AI 状态始终在自己的状态灯中显示，不能因为 Go 状态更严重而隐藏。

来源规则：正常已安装应用显示 `已保存到此设备` 或 `未配置`，而不是“读取 shell 环境”；仅当受显式开发/CI/UAT 标记的本次会话实际覆盖时显示 `测试会话覆盖（未保存）`。用户清除后显示 `未配置`，不允许重启后悄然变成环境来源。

## 关键界面与交互

### 1. `Go 服务` 右侧面板

布局从上至下固定为：

1. **当前状态卡片**：标题 `Go 服务`；展示配置来源、配置状态、三个字段的“已配置/需填写/格式无效”汇总、固定短指纹（例如 `配置指纹 · cfg_7F3A`）和 readiness。没有完整配置时不得显示指纹。
2. **运行状态与诊断**：展示 `sidecar`、`端口`、`健康检查`、`依赖 readiness` 四个带标签行。若 Go 是 degraded，必须附说明“Go 服务未完全就绪；Agent 仍可继续使用。”
3. **配置表单**：依次为 `数据库 URL`、`Redis URL`、`JWT 密钥`。三个字段都使用空的可编辑控件；已有值只显示“已配置”及固定短指纹。JWT 使用密码输入类型和“显示/隐藏”按钮（若实现），切换绝不改变存储值。
4. **操作区**：主按钮 `保存配置`；并列次操作 `测试连接`（已有保存配置时）或 `测试当前配置`（尚未保存、针对表单值）；明确的 `从环境变量导入`；危险次操作 `清除本地配置`。
5. **结果区**：保存、导入或测试的最新结果，使用有文本的 inline alert/card，不只依赖 toast。Toast 仅作短暂确认，不能是唯一反馈。

默认焦点：由 StatusBar “打开设置”、诊断 CTA 或显式配置入口打开时，焦点进入第一个缺失/无效字段；若配置完整则落在 `数据库 URL` 字段。关闭面板后，焦点回到触发它的 CTA 或状态按钮。

**保存与测试分离：**

- `保存配置` 只在三个字段均非空且本地格式校验通过时可提交，写入完整三字段；网络不可达不得阻止保存。
- `测试连接` 或 `重试 readiness` 不会保存未提交的表单，也不会更改配置来源；它只报告当前已保存配置或明确标识为“当前未保存输入”的测试结果。
- 测试期间显示按钮加载态、`正在检查 Go 服务与依赖…`，禁用相同动作；不禁用 Agent、聊天或其他面板。
- 测试成功显示 `Go 服务依赖可用`；失败依照诊断类别显示下一步动作，仍保留保存配置。

**环境导入：**

- 正常安装态，入口始终可见但只有在完整有效 DB/Redis/JWT 环境组存在时可执行；没有可导入组时显示 `未发现可完整导入的环境配置。请填写三项配置后保存。`，不创建部分配置。
- 执行前使用非破坏性确认，文案：`从环境变量导入会保存当前完整的 Go 配置到此设备。不会显示或复制其中的值。是否继续？`
- 成功后显示 `已从环境变量导入并保存到此设备。`；失败时逐字段说明 `数据库 URL 未提供或无效`、`Redis URL 未提供或无效`、`JWT 密钥未提供或无效`。
- 不显示变量名称对应的实际值；不得把环境导入称为常规启动要求。

**清除：**

- `清除本地配置` 使用 `AlertDialog`，仅删除本地持久配置，不影响本次受标记会话覆盖。
- 确认标题：`清除 Go 服务本地配置？`；正文：`这会移除保存在此设备上的数据库 URL、Redis URL 和 JWT 密钥。Go 服务将进入未配置的降级状态；Agent 仍可继续使用。`
- 按钮：取消 `保留配置`，破坏性确认 `清除配置`。
- 完成后清空三个输入控件，状态卡片变为 `未配置`，显示 `本地 Go 配置已清除。可重新填写，或显式从环境变量导入。`；绝不自动导入。

### 2. StatusBar 与 Popover

现有 Go 状态灯保持 24px 高度和 `ProcessLight` 密度。文字格式采用以下优先级：

| 情况 | 状态栏文本 | Popover 必含内容与动作 |
|---|---|---|
| 未配置 | `Go · 未配置 · 降级` | `尚未保存完整的 Go 服务配置。Agent 仍可继续使用。`；`打开设置`、`从环境变量导入` |
| 单字段缺失/无效 | `Go · 配置不完整 · 降级` | 字段级文字诊断；`打开设置` |
| 会话覆盖 | `Go · 测试会话 · {运行中/降级}` | `当前使用仅本次启动的测试会话配置，未保存到此设备。`；`打开设置`、`测试连接` |
| 已配置且 checking | `Go · 托管 · 检查中` | 来源、`正在检查 readiness`；`查看日志` |
| 已配置且 ready | `Go · 托管 · 可用` 或 `Go · 外部 · 可用` | 状态/所有权/健康、配置指纹；`打开设置`、`测试连接`、`查看日志` |
| 依赖未就绪 | `Go · 托管 · 降级` | `Go 服务已启动，但依赖未完全就绪；Agent 仍可继续使用。`；诊断类别、`重试 readiness`、`查看日志` |
| 端口冲突 | `Go · 冲突` | `本机已有进程占用了 Paladin 需要的端口，但未通过健康检查。`；`重新检测`、`查看日志` |
| sidecar 失败 | `Go · 启动失败` | 脱敏后的固定类别和诊断摘要；托管时 `重启`/`重新检测`，外部时只允许 `重新检测`；`查看日志` |

Popover 操作顺序固定：`打开设置` → `测试连接/重试 readiness`（适用时） → `重启`（仅托管且适用） → `重新检测` → `查看日志`。外部 owner 的 `重启`、`停止`必须禁用，并保留原因文字。原始 `last_error`、stderr 和日志必须先通过中心 redactor；UI 只展示脱敏摘要，不展示环境变量值、DSN、Redis URL、JWT、token、API key、仓库路径或 shell 命令。

### 3. 空态、加载态和错误态

| 状态 | 必须文案 | 可用操作 |
|---|---|---|
| 空态/首次使用 | 标题：`尚未配置 Go 服务`；正文：`填写数据库 URL、Redis URL 和 JWT 密钥后，可启用可选的 Go 服务功能。未配置时，Agent 仍可继续使用。` | `保存配置`；`从环境变量导入` |
| 加载配置 | `正在读取此设备上的 Go 服务配置…` | 骨架或紧凑 spinner；避免闪现“未配置” |
| 保存中 | `正在保存配置…` | 禁用重复保存；不禁用测试以外的全局工作流 |
| 依赖检查中 | `正在检查 Go 服务与依赖…` | 禁用重复检查；允许打开日志和继续聊天 |
| 数据库不可用 | `数据库连接不可用。配置已保留；请检查数据库服务和连接信息后重试。` | `重试 readiness`、`打开设置` |
| Redis 不可用 | `Redis 连接不可用。配置已保留；请检查 Redis 服务和连接信息后重试。` | `重试 readiness`、`打开设置` |
| JWT 缺失/无效 | `JWT 密钥缺失或无效。请在 Go 服务设置中填写有效密钥后保存。` | `打开设置` |
| sidecar/端口/健康问题 | 使用结构化类别对应的固定中文说明，不含内部路径或命令 | `重新检测`/`重启`/`查看日志`，视 owner 而定 |

当 AI provider 同时未配置时，AI 状态仍使用既有 `尚未配置 AI provider` 和 `配置 AI provider` 文案；不得在 Go 面板中暗示填写 DB/Redis/JWT 可以解决 AI 对话不可用问题。

## 文案与安全契约

| 元素 | 固定文案/规则 |
|---|---|
| 主 CTA | `保存配置` |
| 次 CTA | `测试连接`、`重试 readiness`、`从环境变量导入`、`打开设置` |
| 配置来源 | `已保存到此设备`、`未配置`、`测试会话覆盖（未保存）`；避免 `shell`、`wrapper` 或内部变量细节 |
| 遮罩状态 | `数据库 URL 已配置 · cfg_7F3A`、`Redis URL 已配置 · cfg_7F3A`、`JWT 密钥已配置 · cfg_7F3A`；指纹必须是固定、不可逆短标识，不能源自 secret 前缀/后缀/长度 |
| 复制 | 所有“复制诊断/复制状态”（若实现）只能复制固定类别、状态、owner、health、非敏感时间戳和下一步动作；禁止复制输入框内容、遮罩原文、DSN/Redis/JWT、环境变量名或原始日志 |
| 错误语气 | 直接、可操作、不归咎用户；不说“请运行 uv/go/pnpm/cargo”、不要求编辑仓库文件、不将 Go 降级称为“Paladin 无法使用” |

不得提供“显示密钥”、“复制密钥”、“导出配置”、“自动安装数据库/Redis”、“保存部分字段”或“在正常 GUI 启动时读取 shell 环境”等功能入口。不得在 console、toast、调试属性、测试快照、截图文案或无障碍标签中泄露机密。

## 可访问性

1. 所有状态均以文字 + 颜色表达；状态点不能单独传达含义。
2. 每个输入均有可见 `<label>`、`id` 与关联帮助/错误文本；字段错误通过 `aria-describedby` 关联，提交失败后焦点移至第一个无效字段。
3. 密码控制、状态灯、打开日志、导入、重试、关闭面板等 icon-only 控件都具备中文 `aria-label` 与 `title`。
4. Popover 用键盘可打开、关闭和遍历；关闭后焦点回到触发的 StatusBar 按钮。`AlertDialog` 遵循现有焦点锁定并提供明确取消按钮。
5. 加载状态通过文本和适当的 `aria-live="polite"` 宣告；保存/测试/导入的成功和失败结果必须以文本出现，不能只依赖 sonner toast。
6. 诊断摘要可换行、滚动和选择文本；敏感字段永远不进入可访问名称或 live region。
7. 主要操作按钮保持可辨识焦点环，警告/成功文字在浅色和深色主题下均使用现有高对比 token。

## 验收标准（可验证）

- [ ] RightPanel 出现独立的 `Go 服务` 标签，位于 `AI Provider` 后；在 300px 宽度可填写、保存、测试、导入和清除，且不新增应用路由或依赖。
- [ ] 读接口与 UI 从不返回或显示 DB URL、Redis URL、JWT 原值、长度、前后缀或可逆掩码；仅显示 presence、来源和固定短指纹。
- [ ] 保存操作要求完整三字段，但依赖不可达时保存仍成功；测试/readiness 结果独立显示，且不把未保存测试值静默持久化。
